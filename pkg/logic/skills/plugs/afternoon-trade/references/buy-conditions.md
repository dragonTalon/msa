# 午盘买入条件判断规则

## 午盘买入特点

午盘买入更注重：
- 早盘信息的验证
- 午间新闻的解读（须通过信息质量门）
- 回调机会的把握
- **市场状态的午盘更新（新增）**

**重要**: 仓位上限和追高限制不再固定，须引用 market-regime-classifier.md 的动态参数。

---

## 0. 前置: 午盘市场状态更新（新增）


## 1. 资金条件

### 基础检查
| 条件 | 要求 |
|------|------|
| 可用余额 | > 0 |
| 已用仓位 | 早盘操作后的最新状态 |
| 剩余可仓位 | 不超过总资产 [动态上限] |

### 买入金额计算
```
max_qty = floor(可用余额 / (价格 × 100)) × 100
实际买入 = min(max_qty, 计划仓位 × 动态首次买入比例)
```

## 3. 午间新闻影响

### 信息质量门（新增）
午间新闻必须通过信息质量门过滤：
```
→ 加载 trading-common/references/news-quality-gate.md
→ 加载 trading-common/references/search-query-tactics.md (多角度搜索)
→ 对午间新闻执行 4 步过滤（来源/时效/拥挤度/完整性）
→ 仅使用通过质量门的信息作为决策依据
```

### 利好消息类型
| 消息类型 | 买入判断 | 质量门要求 |
|----------|----------|-----------|
| 行业利好 | 关注相关板块龙头 | 需 ≥ A 级来源 |
| 公司利好 | 关注具体公司 | 需 ≥ B 级来源 + 交叉验证 |
| 政策利好 | 关注受益板块 | 需 ≥ A 级来源 |

### 新闻处理流程
```
1. 多角度搜索午间新闻（3 组并行查询）
2. 通过信息质量门过滤
3. 检查拥挤度（板块涨幅 > 3% = crowded）
4. 识别利好/利空消息
5. 筛选相关股票
6. 结合技术面判断
```

## 4. 技术信号

### 午盘特殊信号
| 信号 | 说明 |
|------|------|
| 早盘回调至支撑位 | 买入机会 |
| 午后放量启动 | 跟进信号 |
| 突破早盘高点 | 趋势确认 |

### 买入时机
- 早盘关注的股票回调至预期价位
- 午间有利好且技术面配合
- 量价配合良好
- **必须通过信息质量门的拥挤度检查**

| 市场状态 | 正常窗口 | 谨慎窗口 | 禁止新开仓 |
|----------|----------|----------|-----------|
| 强牛/牛/弱牛 | 13:00-14:30 | 14:00-14:30 | 14:30 后 |

| 弱熊/熊 | 13:00-13:30 | 13:30-14:00 | 14:00 后 |
| 强熊 | — | — | 全天不建议新开仓 |

**熊市时间窗口缩窄**: 弱熊/熊市中新开仓窗口缩短至 13:00-13:30，强熊中不建议任何新开仓。

## 午盘买入限制

| 限制 | 说明 |
|------|------|
| 时间限制 | 根据市场状态动态调整（见上表） |
| 仓位限制 | 首次买入不超过计划仓位 [动态比例] |
| 追高限制 | 涨幅 > [动态阈值] 不买入 |
| 拥挤度限制 | 板块涨幅 > 3%（牛市 > 5%）不买入 |
| 参考早盘 | 优先关注早盘提及的标的 |
| 质量门 | 午间新闻必须通过信息质量门 |

**注意**: 上述 [动态] 标记的参数均需从 market-regime-classifier.md 获取，不使用固定值。
