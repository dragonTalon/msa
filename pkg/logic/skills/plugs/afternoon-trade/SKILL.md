# Afternoon Trade - 午盘操作

## 基本信息

| 属性 | 值 |
|------|-----|
| 触发时间 | 13:00 - 14:30 |
| 操作倾向 | **以买入为主，卖出为辅** |
| 执行方式 | **必须实际执行交易** |

---

## 【强制前置】

**第一步必须调用 `read-error` skill！**

```
⚠️ 在执行任何操作前，必须先读取历史错误经验。
```

---

## 执行流程

### Phase 1: 信息收集（强制执行）

#### Step 1: 【强制】读取历史错误
```
调用 read-error skill
→ 将错误经验拼接到后续分析的prompt中
```

#### Step 2: 搜索当日午间新闻
```
调用 web_search("今日午间财经新闻 A股")
→ 选取1-5个最相关链接
调用 fetch_page_content 获取完整内容
→ 分析新闻对市场的影响
→ 识别可能的利好/利空板块
```

#### Step 3: 读取早盘Session
```
调用 query_sessions_by_date(date="today", tag="morning-session")
→ 提取早盘的：
  - 操作决策和执行结果
  - 预测和建议
  - 关注要点
→ 对比早盘预测与上午实际走势
```

#### Step 4: 读取用户画像和策略
```
读取 .msa/knowledge/user-profile/
读取 .msa/knowledge/strategies/
读取 .msa/knowledge/insights/
```

#### Step 5: 获取当前账户状态
```
调用 get_account_summary → 可用余额
调用 get_positions → 当前持仓（早盘操作后的最新状态）
调用 get_transactions(今日) → 今日已执行交易
```

### Phase 2: 分析决策

#### 早盘回顾与验证
```
对比早盘预测与实际走势：
- 早盘判断是否正确？
- 建议的操作是否已执行？
- 市场走势是否符合预期？
- 是否需要调整策略？
```

#### 买入决策（主要）

**买入条件判断：**
1. 可用资金是否充足？
2. 早盘关注的股票是否出现买入机会？
3. 午间新闻是否有利好消息？
4. 技术面是否出现买入信号？
5. 是否符合用户策略的买入规则？

**买入决策：**
```
买入决策：
- 目标代码: sz000858
- 目标名称: 五粮液
- 建议买入: [根据可用资金计算]
- 买入原因: 午盘回调至支撑位 / 利好消息刺激 / 技术面金叉
```

#### 卖出决策（次要）

**如果早盘未执行的卖出，评估是否仍需执行：**
1. 卖出理由是否仍然成立？
2. 是否有新的卖出理由？
3. 当前价格是否更优？

---

### Phase 3: 实际执行操作（强制执行）

**重要：先输出决策，再自动执行，不需要用户确认**

#### 执行买入（主要）

```
FOR each 买入决策:
  1. 调用 get_account_summary 确认可用余额
  2. 调用 get_stock_quote(stock_code) 获取当前价格
  3. 计算最大可买数量（整手）:
     max_qty = floor(可用余额 / (价格 × 100)) × 100
     所需金额 = max_qty × 价格 + 5 (手续费)
  4. 检查金额是否充足:
     如果可用余额 < 所需金额:
       输出 "【买入放弃】[股票名称] 金额不足，可用余额 [xxx]元，需要 [xxx]元"
       跳过此买入
  5. 输出决策：
     "【买入执行】准备买入 [股票名称]([股票代码]) [数量]股，当前价 [价格]元，原因：[原因]"
  6. 调用 submit_buy_order:
     - stock_code: 股票代码
     - stock_name: 股票名称
     - quantity: 买入数量
     - price: 当前价格
     - fee: 5.0 (默认手续费)
  7. 输出执行结果
  8. 记录到当前Session
```

#### 执行卖出（如有）

```
FOR each 卖出决策:
  1. 调用 get_positions 确认持仓
  2. 调用 get_stock_quote 获取价格
  3. 确认卖出数量 <= 持仓数量
  4. 输出决策："【卖出执行】准备卖出..."
  5. 调用 submit_sell_order 执行
  6. 输出执行结果
  7. 记录到当前Session
```

## 约束条件

1. **禁止透支**：金额不足必须放弃买入
2. **时间敏感**：14:30后不建议新开仓
3. **参考早盘**：必须读取早盘Session的决策
4. **执行不确认**：直接执行，不需要确认
5. **错误必读**：必须先执行 read-error skill
6. **分段买入**：参考 trading-common 的仓位管理规则，首次买入不超过计划仓位的 50%
