# 买入条件判断规则

## 买入决策框架

**早盘买入原则：谨慎为主，以下条件需同时满足多个**

---

## 1. 资金条件

### 基础检查
| 条件 | 要求 |
|------|------|
| 可用余额 | > 0 |
| 单只股票仓位 | 不超过总资产 30% |
| 整体仓位 | 不超过总资产 80% |

### 买入金额计算
```
max_qty = floor(可用余额 / (价格 × 100)) × 100
实际买入 = min(max_qty, 计划仓位 × 50%)  // 首次买入不超过计划仓位的 50%
```

---

## 2. 标的选择

### 来源优先级
1. 昨日笔记中的关注股票
2. 用户策略中关注的标的
3. 早盘新闻提到的机会
4. 技术面出现买入信号的股票

### 标的筛选
| 条件 | 要求 |
|------|------|
| 涨幅 | < 3%（不追高） |
| 流动性 | 换手率 > 1% |
| 估值 | PE 不高于行业平均 20% |

---

## 3. 技术信号

### 看涨信号
| 信号 | 权重 |
|------|------|
| 股价站上 5 日均线 | 低 |
| MACD 金叉 | 中 |
| RSI 从超卖区回升 | 中 |
| 放量突破阻力位 | 高 |
| 布林带下轨反弹 | 低 |

### 买入时机
- 至少 2 个技术信号共振
- 量价配合良好
- 未出现明显的卖出信号

---

## 4. 策略规则买入

### 用户策略匹配
检查用户画像中的买入规则：
- 用户风险偏好对应的仓位限制
- 用户投资风格对应的选股标准
- 用户行业偏好

---

## 买入数量决策

| 情况 | 建议买入比例 |
|------|--------------|
| 首次买入 | 计划仓位的 50% |
| 加仓（盈利中） | 剩余计划仓位的 50% |
| 加仓（趋势确认） | 剩余全部 |

---

## 买入执行检查

执行买入前必须确认：
```
□ 可用余额 >= 买入金额 + 手续费
□ 股票涨幅 < 3%（不追高）
□ 买入数量为 100 的整数倍
□ 符合用户策略规则
```

---

## 早盘买入限制

| 限制 | 说明 |
|------|------|
| 时间限制 | 11:00 后谨慎新开仓 |
| 仓位限制 | 首次买入不超过计划仓位 50% |
| 追高限制 | 涨幅 > 3% 不买入 |
| 单日限制 | 同一股票累计不超过计划仓位 70% |
