# 买入条件判断规则

## 买入决策框架

**早盘买入原则：谨慎为主，以下条件需同时满足多个**

**重要**: 仓位上限和追高限制不再固定，须引用 market-regime-classifier.md 的动态参数。

---

## 0. 前置: 加载市场状态（新增）

在判断买入条件前，必须先确定当前市场状态：

```
→ 加载 trading-common/references/market-regime-classifier.md
→ 加载 trading-common/references/market-regime-decision-logic.md
→ 确定当前市场状态和对应的参数
```

## 1. 资金条件

### 基础检查
| 条件 | 要求 |
|------|------|
| 可用余额 | > 0 |
| 单只股票仓位 | 不超过总资产 [动态比例]（见上表） |
| 整体仓位 | 不超过总资产 [动态上限] |

### 买入金额计算
```
max_qty = floor(可用余额 / (价格 × 100)) × 100
实际买入 = min(max_qty, 计划仓位 × 动态首次买入比例)
```

## 2. 标的选择

### 来源优先级
1. 昨日笔记中的关注股票
2. 用户策略中关注的标的
3. 早盘新闻提到的机会（须通过信息质量门）
4. 技术面出现买入信号的股票

### 标的筛选
| 条件 | 要求 |
|------|------|
| 涨幅 | < [动态追高阈值]（见上表） |
| 流动性 | 换手率 > 1% |
| 估值 | PE 要求视市场状态而定（见下表） |

### PE 估值容忍度（动态）

| 市场状态 | PE 要求 |
|----------|--------|
| 强牛/牛 | 不高于行业均值 30% |
| 弱牛/震荡偏牛 | 不高于行业均值 20% |
| 震荡偏熊 | 不高于行业均值 10% |
| 弱熊/熊/强熊 | 必须低于行业均值 |

## 3. 技术信号

### 看涨信号
| 信号 | 权重 |
|------|------|
| 股价站上 5 日均线 | 低 |
| MACD 金叉 | 中 |
| RSI 从超卖区回升 | 中 |
| 放量突破阻力位 | 高 |
| 布林带下轨反弹 | 低 |

### 买入时机
- 牛市: 至少 1 个强技术信号即可
- 震荡: 至少 2 个技术信号共振
- 熊市: 至少 3 个技术信号共振 + 基本面支撑
- 量价配合良好
- 未出现明显的卖出信号

## 买入数量决策

| 情况 | 建议买入比例 |
|------|--------------|
| 首次买入 | 计划仓位的 50%（牛市中 60%） |
| 加仓（盈利中） | 剩余计划仓位的 50% |
| 加仓（趋势确认） | 剩余全部 |

---

## 买入执行检查

执行买入前必须确认：
```
□ 可用余额 >= 买入金额 + 手续费
□ 股票涨幅 < [动态追高阈值]
□ 买入数量为 100 的整数倍
□ 符合用户策略规则
□ 信息质量门已通过
□ 推理链已输出
```

---

## 早盘买入限制

| 限制 | 说明 |
|------|------|
| 时间限制 | 11:00 后谨慎新开仓 |
| 仓位限制 | 首次买入不超过计划仓位 [动态比例] |
| 追高限制 | 涨幅 > [动态阈值] 不买入 |
| 单日限制 | 同一股票累计不超过计划仓位 70% |

**注意**: 上述 [动态] 标记的参数均需从 market-regime-classifier.md 获取，不使用固定值。

## Phase 2 增强规则 (v2.3.0)

### 操作类型选择

买入前必须参照 `trading-common/references/operation-types.md` 选择操作类型:

1. 根据信号数量和质量判断信号强度
2. 查"信号强度 → 操作映射"表确定操作类型
3. 根据置信度确定仓位系数
4. 最终仓位 = 计划仓位 × 仓位系数

### 持仓相关性检查

买入前必须加载并执行 `trading-common/references/correlation-checklist.md`:

1. 调用 `get_stock_industry` 获取新标的行业分类
2. 与现有持仓行业对比，计算行业暴露
3. 检查概念板块集中度
4. 如触发限制 → 调整仓位或输出 HOLD

### ATR 动态止损

买入时必须加载 `trading-common/references/risk-management.md`:
- 调用 `get_stock_history_k` 获取 14 天日K线
- 计算 ATR(14) 和初始止损位（入场价 - 2×ATR）
- 将止损位记录在推理链中

### 买入执行检查清单（更新）

在原有检查清单基础上增加:
```
□ 已选择操作类型 (TRIAL/STANDARD/ADD)
□ 已标注置信度（高/中/低）及仓位系数
□ 已通过行业集中度检查（单一行业≤50%、子行业≤30%、概念≤20%）
□ 已计算 ATR 初始止损位（参考 risk-management.md）
□ 已检查回撤熔断状态（日内≤5%、周内≤10%）
```
