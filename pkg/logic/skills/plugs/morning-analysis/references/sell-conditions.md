# 卖出条件判断规则

## 卖出决策框架

对每只持仓进行以下检查，满足任一条件则考虑卖出：

---

## 1. 止盈卖出（优先级最高）

### 条件判断
| 条件 | 触发动作 |
|------|----------|
| 盈利超过目标收益率（默认 +8%） | 卖出 50% 锁定收益 |
| 盈利超过目标收益率 1.5 倍 | 卖出全部仓位 |
| 达到预设止盈价位 | 按计划卖出 |

### 执行方式
- 分批止盈：首次卖出 50%，剩余观察
- 一次性止盈：盈利超过 15% 时可考虑全部卖出

---

## 2. 技术面走弱

### 条件判断
| 条件 | 说明 |
|------|------|
| 跌破支撑位 | 跌破关键支撑价位 |
| 均线死叉 | 5日均线跌破10日均线 |
| 放量下跌 | 成交量放大且股价下跌 |
| MACD 死叉 | DIF 下穿 DEA |

### 执行方式
- 技术走弱信号出现：卖出 30%-50%
- 多个信号共振：清仓

---

## 3. 资金流出

### 条件判断
| 条件 | 说明 |
|------|------|
| 主力连续流出 | 连续 3 日主力净流出 |
| 北向资金减持 | 北向资金连续减持 |
| 大单卖出 | 出现大单主动卖出 |

### 执行方式
- 资金流出明显：减仓 50%
- 资金大幅流出：清仓

---

## 4. 负面消息

### 条件判断
| 消息类型 | 影响 |
|----------|------|
| 公司负面 | 业绩下滑、高管变动、监管处罚 |
| 行业负面 | 政策利空、行业周期下行 |
| 宏观负面 | 利率上升、经济衰退信号 |

### 执行方式
- 重大负面：立即清仓
- 一般负面：减仓观察

---

## 5. 策略规则卖出

### 用户策略匹配
检查用户画像中的卖出规则：
- 用户风险偏好对应的止损位
- 用户投资周期对应的止盈目标
- 用户行业偏好对应的调仓需求

---

## 卖出数量决策

| 情况 | 建议卖出比例 |
|------|--------------|
| 触及止盈位 | 50% |
| 多个卖出信号共振 | 100% |
| 单一技术信号 | 30% |
| 负面消息 | 50%-100% |

---

## 卖出执行检查

执行卖出前必须确认：
```
□ 卖出数量 <= 持仓数量
□ 当前价格已获取
□ 卖出原因已记录
```
