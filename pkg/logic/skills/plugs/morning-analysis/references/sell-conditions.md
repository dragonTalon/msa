# 卖出条件判断规则

## 卖出决策框架

对每只持仓进行以下检查，满足任一条件则考虑卖出：

---

## 1. 止盈卖出（优先级最高）

### 条件判断
| 条件 | 触发动作 |
|------|----------|
| 盈利超过目标收益率（默认 +8%） | 卖出 50% 锁定收益 |
| 盈利超过目标收益率 1.5 倍 | 卖出全部仓位 |
| 达到预设止盈价位 | 按计划卖出 |

### 执行方式
- 分批止盈：首次卖出 50%，剩余观察
- 一次性止盈：盈利超过 15% 时可考虑全部卖出

---

## 2. 技术面走弱

### 条件判断
| 条件 | 说明 |
|------|------|
| 跌破支撑位 | 跌破关键支撑价位 |
| 均线死叉 | 5日均线跌破10日均线 |
| 放量下跌 | 成交量放大且股价下跌 |
| MACD 死叉 | DIF 下穿 DEA |

### 执行方式
- 技术走弱信号出现：卖出 30%-50%
- 多个信号共振：清仓

---

## 3. 资金流出

### 条件判断
| 条件 | 说明 |
|------|------|
| 主力连续流出 | 连续 3 日主力净流出 |
| 北向资金减持 | 北向资金连续减持 |
| 大单卖出 | 出现大单主动卖出 |

### 执行方式
- 资金流出明显：减仓 50%
- 资金大幅流出：清仓

---

## 4. 负面消息

### 条件判断
| 消息类型 | 影响 |
|----------|------|
| 公司负面 | 业绩下滑、高管变动、监管处罚 |
| 行业负面 | 政策利空、行业周期下行 |
| 宏观负面 | 利率上升、经济衰退信号 |

### 执行方式
- 重大负面：立即清仓
- 一般负面：减仓观察

---

## 5. 策略规则卖出

### 用户策略匹配
检查用户画像中的卖出规则：
- 用户风险偏好对应的止损位
- 用户投资周期对应的止盈目标
- 用户行业偏好对应的调仓需求

---

## 卖出数量决策

| 情况 | 建议卖出比例 |
|------|--------------|
| 触及止盈位 | 50% |
| 多个卖出信号共振 | 100% |
| 单一技术信号 | 30% |
| 负面消息 | 50%-100% |

---

## 卖出执行检查

执行卖出前必须确认：
```
□ 卖出数量 <= 持仓数量
□ 当前价格已获取
□ 卖出原因已记录
```

## Phase 2 增强规则 (v2.3.0)

### 操作类型选择

卖出前必须参照 `trading-common/references/operation-types.md`:

1. 根据卖出信号强度查映射表确定操作类型
2. SELL_REDUCE — 单一信号走弱，减持 30%
3. SELL_HALF — 盈利达标或多信号走弱，减仓 50%
4. SELL_ALL — 触发止损或多信号共振，清仓 100%

### ATR 移动止盈

加载 `trading-common/references/risk-management.md`，计算移动止损位:
- 对于已有盈利的持仓，止损位上移至: 最高价 - 1×ATR(14)
- 止损位只能上移不能下移 (ratchet 机制)
- 当前价触及移动止损位 → 执行 SELL_ALL

### 时间止损

- 持仓 N 日未达预期收益: 减半仓（N=5天，短线默认）
- 持仓 2N 日仍亏损: 清仓

### 卖出执行检查清单（更新）

在原有检查清单基础上增加:
```
□ 已选择卖出操作类型 (REDUCE/HALF/ALL)
□ 已检查 ATR 移动止损位（盈利持仓必须）
□ 已标注置信度
□ 已检查时间止损（持仓天数）
```
