# Morning Analysis - 早盘开市操作

## 基本信息

| 属性 | 值                  |
|------|--------------------|
| 触发时间 | 9:30-11:30（股票操作阶段） |
| 操作倾向 | **以卖出为主，买入为辅**     |
| 执行方式 | **必须实际执行交易**       |

---

## 【强制前置】

**第一步必须调用 `read-error` skill！**

```
⚠️ 在执行任何操作前，必须先读取历史错误经验。
```

---

## 执行流程

### Phase 1: 信息收集（强制执行）

#### Step 1: 【强制】读取历史错误
```
调用 read-error skill
→ 将错误经验拼接到后续分析的prompt中
```

#### Step 2: 读取用户笔记
```
读取 .msa/knowledge/notes/YYYY-MM-DD.md（昨日笔记）
→ 如果不存在，记录"无昨日笔记"
→ 提取：关注股票、市场观点、操作计划
```

#### Step 3: 读取用户画像和策略
```
读取 .msa/knowledge/user-profile/
→ 了解用户风险偏好、投资风格、关注领域

读取 .msa/knowledge/strategies/
→ 获取用户的交易策略规则

读取 .msa/knowledge/insights/
→ 获取市场洞察信息
```

#### Step 4: 读取上一交易日总结
```
读取 .msa/knowledge/summaries/ 最近一个工作日的文件
→ 获取上一交易日的复盘信息
→ 获取"明日关注"要点
```

#### Step 5: 获取账户状态
```
调用 get_account_summary → 总资产、可用余额、持仓市值
调用 get_positions → 当前持仓详情（股票、数量、成本、盈亏）
调用 get_transactions(limit=10) → 最近交易记录
```

---

### Phase 2: 分析决策

综合所有信息，生成操作决策。

#### 卖出决策（主要）

对每只持仓分析是否需要卖出：

**卖出条件判断：**
1. 是否触及止盈位？（盈利超过目标）
2. 技术面是否走弱？（跌破支撑、量能萎缩）
3. 是否有资金撤离迹象？
4. 是否有负面消息？
5. 是否符合用户策略的卖出规则？

**如果满足任一条件，生成卖出决策：**
```
卖出决策：
- 持仓代码: sh600519
- 持仓名称: 贵州茅台
- 当前持仓: 100股
- 建议卖出: 50股（减半仓）/ 100股（清仓）
- 卖出原因: 触及止盈位 / 技术面走弱 / 资金撤离
```

#### 买入决策（次要）

**买入条件判断：**
1. 可用资金是否充足？
2. 是否有优质标的？（参考昨日笔记、策略）
3. 技术面是否出现买入信号？
4. 是否符合用户策略的买入规则？

**如果满足条件，生成买入决策：**
```
买入决策：
- 目标代码: sz000858
- 目标名称: 五粮液
- 建议买入: [根据可用资金计算]
- 买入原因: 技术面突破 / 估值合理 / 资金流入
```

---

### Phase 3: 实际执行操作（强制执行）

**重要：先输出决策，再自动执行，不需要用户确认**

#### 执行卖出

```
FOR each 卖出决策:
  1. 调用 get_stock_quote(stock_code) 获取当前价格
  2. 输出决策：
     "【卖出执行】准备卖出 [股票名称]([股票代码]) [数量]股，当前价 [价格]元，原因：[原因]"
  3. 调用 submit_sell_order:
     - stock_code: 股票代码
     - stock_name: 股票名称
     - quantity: 卖出数量
     - price: 当前价格
     - fee: 5.0 (默认手续费)
  4. 输出执行结果
  5. 记录到当前Session
```

#### 执行买入

```
FOR each 买入决策:
  1. 调用 get_account_summary 确认可用余额
  2. 调用 get_stock_quote(stock_code) 获取当前价格
  3. 计算最大可买数量（整手）:
     max_qty = floor(可用余额 / (价格 × 100)) × 100
     所需金额 = max_qty × 价格 + 5 (手续费)
  4. 检查金额是否充足:
     如果可用余额 < 所需金额:
       输出 "【买入放弃】[股票名称] 金额不足，可用余额 [xxx]元，需要 [xxx]元"
       跳过此买入
  5. 输出决策：
     "【买入执行】准备买入 [股票名称]([股票代码]) [数量]股，当前价 [价格]元，原因：[原因]"
  6. 调用 submit_buy_order:
     - stock_code: 股票代码
     - stock_name: 股票名称
     - quantity: 买入数量
     - price: 当前价格
     - fee: 5.0 (默认手续费)
  7. 输出执行结果
  8. 记录到当前Session
```

## 约束条件

1. **禁止透支**：可用余额不足时必须放弃买入
2. **禁止卖空**：卖出数量不能超过持仓数量
3. **整手交易**：买入数量必须是100的整数倍
4. **执行不确认**：输出决策后直接执行，不需要用户确认
5. **错误必读**：必须先执行 read-error skill
