# 推理质量审查标准

## 概述

复盘时不仅评估操作结果，还要评估决策推理过程的质量。区分"正确决策运气不好"与"错误决策运气好"。

---

## 推理质量评级 (A-E)

### A 级: 优秀推理
```
标准:
  - 6 段推理链完整输出
  - 触发条件判断准确
  - 理由详述充分(支持≥2, 反对≥1)
  - 反向因素检查全部通过
  - 信息质量评估准确
  - 市场状态判断正确
  - 决策逻辑与市场状态一致
  - 预期结果明确(止损/止盈/持仓期)

结果无关: 即使最终亏损，A 级推理仍值得保留为正向经验
```

### B 级: 良好推理
```
标准:
  - 6 段推理链完整
  - 主要理由充分
  - 反向检查基本通过
  - 信息质量和市场状态有引用
  - 决策可执行

缺失: 部分细节不够深入(如反对理由不够全面, 反向检查不够严格)
```

### C 级: 及格推理
```
标准:
  - 推理链基本完整(可能缺 1-2 段)
  - 有支持理由但缺乏反对理由
  - 反向检查草率
  - 信息质量和市场状态引用不完整
  - 决策勉强可执行

风险: C 级推理的决策执行风险较高, 需要特别关注结果
```

### D 级: 不合格推理
```
标准:
  - 推理链严重不完整(缺 ≥ 3 段)
  - 仅有结论无推导过程 ← 对应 C3 缺陷
  - 决策依据模糊("感觉""可能""应该")
  - 无信息质量评估
  - 无市场状态考量
  - 无止损止盈设置

处理: D 级推理视为情绪化决策, 必须记录到 errors/
```

### E 级: 无推理
```
标准:
  - 完全没有推理链
  - 仅有操作指令("买入 XX 100股")
  - 无法追溯任何决策逻辑 ← C3 缺陷的最严重表现

处理: E 级推理 = 严重错误, 必须:
  1. 记录到 errors/ (severity: high)
  2. 标记为"情绪化决策"
  3. 在明日操作中优先回顾
```

## "正确决策"vs"错误决策"判定矩阵

| 推理质量 | 结果 | 判定 | 处理 |
|----------|------|------|------|
| A/B 级 | 盈利 | ✅ 正确决策, 好结果 | 复用经验 |
| A/B 级 | 亏损 | ✅ 正确决策, 坏运气 | 保留逻辑, 不否定方法 |
| C 级 | 盈利 | ⚠️ 运气好, 决策有缺陷 | 优化推理, 不过度自信 |
| C 级 | 亏损 | ⚠️ 决策不够严谨 | 补充推理后复用 |
| D 级 | 盈利 | ❌ 错误决策, 好运气 | **危险 — 可能强化坏习惯** |
| D 级 | 亏损 | ❌ 错误决策, 坏结果 | 记录错误, 修正流程 |
| E 级 | 盈利 | ❌ 赌博式操作 | **最危险 — 必须警告** |
| E 级 | 亏损 | ❌ 不可接受的错误 | 严重错误, 升级处理 |

## 审查维度评分

### 每个决策的审查维度 (1-5 分)

| 维度 | 评分标准 |
|------|----------|
| **触发合理性** | 5=触发标准明确且满足, 1=无触发或触发不成立 |
| **理由充分性** | 5=多空理由充分权衡, 1=仅有单方面理由 |
| **反确认偏误** | 5=严格反向检查, 1=未做任何反向搜索 |
| **信息质量意识** | 5=准确评估信息质量并影响决策, 1=忽略信息质量 |
| **市场状态适配** | 5=决策参数与市场状态匹配, 1=无视市场状态 |
| **可执行性** | 5=明确代码/数量/价格/止损止盈, 1=模糊不清 |
| **结果归因** | 5=清楚为什么赚/亏, 1=不知道为什么赚/亏 |

### 维度评分 → A-E 评级聚合规则
```
计算 7 个维度的平均分:
  平均分 ≥ 4.5 且 最低维度 ≥ 3:  A 级 (优秀)
  平均分 ≥ 3.5 且 最低维度 ≥ 2:  B 级 (良好)
  平均分 ≥ 2.5:                  C 级 (及格)
  平均分 ≥ 1.5:                  D 级 (不合格)
  平均分 < 1.5 或 缺 ≥ 3 维度:   E 级 (无推理)

特殊情况:
  - 任一维度为 1 分 → 最高为 C 级
  - 缺少整段推理链 → 直接 D 级
  - 完全无推理链 → 直接 E 级
```

## 复盘推理审查输出

```markdown
## 推理质量审查

### 决策列表
| ID | 类型 | 推理等级 | 结果 | 判定 |
|----|------|----------|------|------|
| dec-001 | buy | B | 盈利+3% | ✅ 正确决策 |
| dec-002 | sell | A | 亏损-1% | ✅ 正确决策, 坏运气 |
| dec-003 | buy | D | 盈利+5% | ❌ 错误决策, 好运气 |

### 推理质量分布
- A 级: [n] 个
- B 级: [n] 个
- C 级: [n] 个
- D 级: [n] 个
- E 级: [n] 个

### 需要改进的决策
[列出 C/D/E 级决策及改进建议]

### 需要警惕的"好运气"
[列出 D/E 级但盈利的决策 — 这些最容易强化坏习惯]

### 需要保留的"坏运气"
[列出 A/B 级但亏损的决策 — 这些不应该否定方法]
```

## 使用方式

1. 在 market-close-summary Step 3 操作复盘时加载本文件
2. 从 Session 中提取每个决策的 decision-log
3. 对比 decision-log 中的推理链与审查标准
4. 对每个决策给出 A-E 评级
5. 使用判定矩阵区分"正确决策"vs"错误决策"
6. 在总结中输出推理质量审查结果
