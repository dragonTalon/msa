# 闭市错误检查清单

## 每日必检项目

**在 Step 4 执行时，逐项检查以下内容：**

---

## 买入错误检查

### 追高买入
```
□ 今日是否有在涨幅 > 3% 时买入？

检查方式：
- 查看今日买入记录
- 检查买入时的股票涨幅
- 涨幅 = (买入价 - 昨收价) / 昨收价 × 100%

如果"是"：
- 严重程度：high
- 必须记录到 errors/ 目录
```

### 涨幅过大追入
```
□ 今日是否有在涨幅 > 50% 时买入？

检查方式：
- 检查买入股票的近期涨幅
- 从高点计算涨幅

如果"是"：
- 严重程度：critical
- 必须记录到 errors/ 目录
```

### 未设止损买入
```
□ 今日买入时是否设置了止损位？

检查方式：
- 查看买入决策记录
- 检查是否有止损价位

如果"否"：
- 严重程度：medium
- 记录提醒
```

### 一次性满仓
```
□ 今日是否单次买入超过计划仓位 70%？

检查方式：
- 计算单次买入金额占比
- 计划仓位 = 总资产 × 30%

如果"是"：
- 严重程度：high
- 必须记录到 errors/ 目录
```

---

## 卖出错误检查

### 未及时止损
```
□ 今日是否有持仓亏损超过 20% 仍未卖出？

检查方式：
- 检查所有持仓的盈亏
- 计算亏损百分比

如果"是"：
- 严重程度：critical
- 必须记录到 errors/ 目录
- 必须在明日优先处理
```

### 过早止盈
```
□ 今日是否有盈利 < 3% 就全部卖出？

检查方式：
- 查看卖出记录
- 检查卖出时的盈利情况

如果"是"：
- 严重程度：low
- 记录分析
```

---

## 执行错误检查

### 漏执行决策
```
□ 今日是否有已决策但未执行的操作？

检查方式：
- 对比决策记录和执行记录
- 找出未执行的决策

如果"是"：
- 严重程度：medium
- 记录原因
```

### 违背策略
```
□ 今日操作是否符合用户策略？

检查方式：
- 读取 user-profile 和 strategies
- 对比今日操作与策略规则

如果"否"：
- 严重程度：medium
- 记录违背的具体内容
```

---

## 情绪化决策检查

```
□ 今日是否有情绪化决策？

判断依据：
- 无明确技术/基本面信号的买卖
- 受新闻影响但未验证的决策
- 与策略不符的冲动操作

如果"是"：
- 严重程度：medium
- 记录情绪化决策的具体情况
```

---

## 检查结果汇总

### 记录模板

如果发现任何错误，使用以下格式记录：

```markdown
---
id: err-XXX
date: YYYY-MM-DD
severity: [critical/high/medium/low]
tags: [错误类型标签]
status: active
---

## 错误描述
[具体发生了什么错误]

## 错误原因
[为什么会犯这个错误]

## 正确做法
[应该怎么做]

## 防范措施
[以后如何避免]

## 相关案例
[今日具体案例]
```

### 无错误时的输出

```
✓ 今日交易检查完成，未发现明显错误。

继续保持良好的交易习惯。
```
