## 交易执行规范

### 执行前检查
1. 调用 `get_account_summary` 确认可用余额
2. 调用 `get_stock_quote` 获取当前价格
3. 计算所需金额：数量 × 价格 + 手续费
4. 确认金额充足，否则放弃操作

### 买入执行流程
```
1. get_account_summary → 获取 available_amt
2. get_stock_quote → 获取 current_price
3. 计算最大可买数量 = floor(available_amt / (price * 100)) * 100  # 整手
4. submit_buy_order(stock_code, stock_name, quantity, price, fee)
```

### 卖出执行流程
```
1. get_positions → 获取持仓数量
2. get_stock_quote → 获取 current_price
3. 确认卖出数量 <= 持仓数量
4. submit_sell_order(stock_code, stock_name, quantity, price, fee)
```

## Session 标签约定

用于区分不同时间段的会话：
- `morning-session` - 早盘会话（9:30-11:30 自动添加）
- `afternoon-session` - 午盘会话（13:00-14:30 自动添加）
- `close-session` - 闭市会话（16:00后 自动添加）

**注意**：Session 标签现在由系统自动添加，无需手动调用 add_session_tag。

## 仓位管理规则

### 计划仓位计算

```
计划仓位 = min(总资产 × 30%, 可用资金)
```

转换为整手数量（100股整数倍）。

### 买入限制参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 单只股票最大仓位 | 总资产 30% | 单只股票持仓上限 |
| 单次买入上限 | 计划仓位 50% | 单次交易最多买入量 |
| 同日累计买入上限 | 计划仓位 70% | 同一交易日同一股票累计 |
| 涨幅限制 | 50% | 涨幅超过此值不追高买入 |

## 分段买入策略

### 方式 A：按时间分段

| 阶段 | 买入比例 | 条件 |
|------|----------|------|
| 首次 | 计划仓位 1/3 | 出现买入信号 |
| 午盘 | 计划仓位 1/3 | 走势符合预期 |
| 次日 | 计划仓位 1/3 | 趋势确认 |

**适用场景**：震荡市、不确定性高

### 方式 B：按价格信号分段

| 阶段 | 买入比例 | 条件 |
|------|----------|------|
| 首次 | 计划仓位 1/3 | 买入信号出现 |
| 加仓 | 计划仓位 1/3 | 上涨 2-3% 或突破关键位 |
| 确认 | 计划仓位 1/3 | 趋势完全确认 |

**适用场景**：趋势明确、有技术支撑

### 方式 C：固定比例分段

| 阶段 | 买入比例 |
|------|----------|
| 第1次 | 40% |
| 第2次 | 40% |
| 第3次 | 20% |

**适用场景**：保守操作、风险控制优先

### 选择依据

查询今日 Session 和历史 errors/summaries，根据以下因素决定：
- 市场环境（震荡/趋势）
- 股票波动性
- 历史类似情况的成功率

## 分段卖出策略

### 正常卖出

- 单次最多卖出：持仓的 50%
- 分批止盈，逐步锁定收益

### 止损卖出（例外情况）

- **条件**：持仓亏损超过 20%
- **操作**：允许一次性全部卖出
- **原则**：止损要果断，不要分批

---

## 约束条件

1. **禁止透支消费**：可用余额不足时必须放弃买入操作
2. **禁止卖空**：卖出数量不能超过持仓数量
3. **整手交易**：买入数量必须是100的整数倍
4. **执行不确认**：输出决策后直接执行，不需要用户确认
5. **分段买入**：首次买入不超过计划仓位的 50%
6. **追高限制**：涨幅超过 50% 不买入
