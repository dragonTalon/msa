# 持仓相关性检查清单 (v2.3.0)

> Phase 2 策略体系升级 — 行业集中度与概念关联风险控制

## 买入前必查

### 1. 行业集中度检查

使用 `get_stock_industry` 查询新标的的申万行业分类（一级和二级），然后与现有持仓对比：

| 维度 | 最大暴露上限 |
|------|-------------|
| 单一申万一级行业 | ≤ 总资产 50% |
| 单一申万二级行业 | ≤ 总资产 30% |
| 单一概念板块 | ≤ 总资产 20% |

**检查流程:**
1. 调用 `get_stock_industry(stock_code)` 获取新标的行业分类（plate 字段）
2. 对每个现有持仓，同样获取其行业分类
3. 按行业 ID 分组计算暴露比例
4. 如有任一维度超限 → 降低仓位或放弃买入

### 2. 概念板块关联检查

使用 `get_board_rank(board_type="02", order=0, count=20)` 获取当前热门概念板块，检查持仓是否同属热门概念。

**检查流程:**
1. 获取前 20 热门概念板块的板块代码
2. 使用板块代码查询成分股，确认持仓股票归属
3. 如同一概念有 ≥3 只持仓或暴露 > 20% → 标注集中风险

### 3. 价格相关性检查 (可选增强)

如果 `get_stock_history_k` 对两只股票都有 ≥60 天数据:
- 取各自收盘价序列
- 计算 Pearson 相关系数 r = cov(X,Y) / (std(X) × std(Y))
- 如果 r > 0.7: 高度相关，降低仓位或放弃
- 如果 r < -0.3: 负相关，可提供对冲效果，适当增加仓位

## 行业集中度计算示例

```
假设总资产 100,000:
- 贵州茅台(sh600519) — 食品饮料/白酒Ⅱ — 市值 15,000 (15%)
- 五粮液(sz000858)   — 食品饮料/白酒Ⅱ — 市值 12,000 (12%)
- 拟买入泸州老窖(sz000568) — 食品饮料/白酒Ⅱ — 计划 10,000 (10%)

食品饮料一级暴露: 15% + 12% + 10% = 37% ≤ 50% ✅
白酒Ⅱ 二级暴露:      15% + 12% + 10% = 37% > 30% ❌

→ 拒绝满仓买入，建议降至 3,000 (3%) 使白酒Ⅱ暴露恰好 30%
```

## 使用说明

1. 每次买入决策前强制执行此检查
2. 检查结果写入推理链的"假设"部分
3. 如触发限制，在"备选方案"中说明调整后的仓位
