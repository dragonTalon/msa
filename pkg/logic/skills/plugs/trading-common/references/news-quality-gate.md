# 信息质量门 - 4步过滤器

## 概述

在 web_search 获取信息后、用于决策前，必须经过此 4 步过滤。防止 AI 追热点导致系统性高位接盘。

---

## Step 1: 来源分类

对每条信息标注来源等级：

| 等级 | 来源类型 | 可信度 | 处理方式 |
|------|----------|--------|----------|
| **S** | 官方公告、财报、交易所披露 | 极高 | 直接采用 |
| **A** | 权威财经媒体 (财新、证券时报、上海证券报) | 高 | 采用，需 1 个交叉验证 |
| **B** | 券商研报、行业分析报告 | 中 | 采用，需 2 个交叉验证 |
| **C** | 财经自媒体、论坛、社交媒体 | 低 | 仅作参考，不单独作为决策依据 |
| **X** | 无法确认来源、匿名消息、传言 | 不可用 | **直接丢弃** |

### 输出格式
```
来源评级：
- S: [数量] 条
- A: [数量] 条
- B: [数量] 条
- C: [数量] 条
- X: [数量] 条 (已丢弃)
综合可信度: [高/中/低]
```

### 综合可信度计算规则
```
高: S ≥ 1 或 A ≥ 3
中: A ≥ 1 或 B ≥ 3
低: 不满足以上（大部分为 C 或存在 X）
```
> 注: 如果 X > 0，综合可信度最高为"中"，因为存在不可用来源说明信息环境嘈杂。

---

## Step 2: 时效检查

判断信息是否已被市场定价：

| 时效 | 状态 | 处理 |
|------|------|------|
| < 30 分钟 | 新鲜 | 重点关注 |
| 30-60 分钟 | 较新鲜 | 正常采用 |
| 60-120 分钟 | 可能已定价 | 谨慎采用，需检查价格反应 |
| > 120 分钟 | 大概率已定价 | 仅作背景参考，不作为买入依据 |

### 已定价判断规则
```
如果 信息发布时间 > 60分钟 且 相关股票/板块涨幅 > 1%:
  → 标记为"已定价"，不作为买入理由
```

---

## Step 3: 拥挤度检查

判断信息是否已被过度交易：

### 板块拥挤度
```
查询相关板块当日涨幅:
  < 1%     → 正常，信息尚未被充分消化
  1%-3%    → 关注，部分资金已进入
  3%-5%    → 拥挤 (crowded)，谨慎追入
  > 5%     → 过度拥挤，禁止追高买入
```

### 个股拥挤度
```
查询相关个股:
  涨幅 > 3% 且 换手率 > 10% → 拥挤，不买入
  涨幅 > 5%                  → 过度拥挤，禁止买入
```

### 牛市中拥挤度阈值调整
```
当市场状态为"强牛/牛/弱牛"时，阈值放宽:
  板块涨幅 > 5% 才算 crowded (vs 默认 3%)
  板块涨幅 > 8% 才算过度拥挤 (vs 默认 5%)
```

---

## Step 4: 完整性检查

判断信息是否足够支撑决策：

| 检查项 | 标准 | 不足时处理 |
|--------|------|-----------|
| 信息条目数 | ≥ 3 条有效信息 | 不足则扩大搜索 |
| 多空平衡 | 至少有 1 条反向观点 | 不足则主动搜索反向意见 |
| 数据完整性 | 有具体数字而非模糊描述 | 模糊信息降级为 C 级 |
| 交叉验证 | 关键信息有 ≥ 2 个来源 | 单一来源信息标注"待验证" |

---

## 质量门输出汇总

完成 4 步过滤后，输出信息质量摘要：

```markdown
## 信息质量摘要

| 维度 | 评级 | 说明 |
|------|------|------|
| 来源可信度 | [高/中/低] | S:[n] A:[n] B:[n] C:[n] X:[n] |
| 时效性 | [新鲜/可接受/已定价] | 最新:[时间] 最旧:[时间] |
| 拥挤度 | [正常/关注/拥挤/过度拥挤] | 板块涨幅:[x%] |
| 完整性 | [充分/可接受/不足] | 有效条目:[n] 反向观点:[有/无] |

综合判定: [可决策 / 需补充信息 / 暂停决策]
```

### 判定规则
- **可决策**: 来源≥中 + 时效≥可接受 + 拥挤度≤关注 + 完整性≥可接受
- **需补充信息**: 任一维度不达标，扩大搜索补充
- **暂停决策**: 来源=低 或 拥挤度=过度拥挤，放弃本轮基于此信息的决策
