# 仓位管理规则

## 计划仓位计算（动态）

**重要**: 仓位比例不再固定为 30%，改为根据市场状态动态调整。

### 加载市场状态
```
在计算仓位前，必须先加载 market-regime-classifier.md 获取当前市场状态。
```

### 动态仓位表

| 市场状态 | 计划仓位比例 | 单只股票上限 | 总仓位上限 |
|----------|-------------|-------------|-----------|
| STRONG_BULL | 总资产 40% | 总资产 40% | 总资产 90% |
| BULL | 总资产 35% | 总资产 35% | 总资产 85% |
| WEAK_BULL | 总资产 30% | 总资产 30% | 总资产 80% |
| RANGING_BULLISH | 总资产 25% | 总资产 25% | 总资产 70% |
| RANGING_BEARISH | 总资产 20% | 总资产 20% | 总资产 60% |
| WEAK_BEAR | 总资产 15% | 总资产 15% | 总资产 50% |
| BEAR | 总资产 10% | 总资产 10% | 总资产 30% |
| STRONG_BEAR | 总资产 5% | 总资产 5% | 总资产 15% |

### 默认值（无法获取市场状态时）
```
计划仓位 = min(总资产 × 30%, 可用资金)  // 保守默认值
```

### 计算公式
```
计划仓位 = min(总资产 × 动态比例, 可用资金)
```

转换为整手数量（100股整数倍）。

## 买入限制参数（动态）

| 参数 | 值 | 说明 |
|------|-----|------|
| 单只股票最大仓位 | 见动态仓位表 | 根据市场状态 |
| 单次买入上限 | 计划仓位 × 动态比例 | 见下方分段策略 |
| 同日累计买入上限 | 计划仓位 70% | 同一交易日同一股票累计 |
| 追高限制 | 见市场状态分类器 | 牛市中放宽，熊市中收紧 |

### 追高限制速查

| 市场状态 | 追高阈值 |
|----------|----------|
| 强牛/牛 | 涨幅 > 5% 不买入 |
| 弱牛/震荡偏牛 | 涨幅 > 3% 不买入 |
| 震荡偏熊/弱熊 | 涨幅 > 2% 不买入 |
| 熊/强熊 | 涨幅 > 1% 不买入 |

## 分段买入策略

### 方式 A：按时间分段

| 阶段 | 买入比例 | 条件 |
|------|----------|------|
| 首次 | 计划仓位 1/3 | 出现买入信号 |
| 午盘 | 计划仓位 1/3 | 走势符合预期 |
| 次日 | 计划仓位 1/3 | 趋势确认 |

**适用场景**：弱牛、震荡偏牛

### 方式 B：按价格信号分段

| 阶段 | 买入比例 | 条件 |
|------|----------|------|
| 首次 | 计划仓位 1/3 | 买入信号出现 |
| 加仓 | 计划仓位 1/3 | 上涨 2-3% 或突破关键位 |
| 确认 | 计划仓位 1/3 | 趋势完全确认 |

**适用场景**：强牛、牛市

### 方式 C：固定比例分段

| 阶段 | 买入比例 |
|------|----------|
| 第1次 | 40% |
| 第2次 | 40% |
| 第3次 | 20% |

**适用场景**：震荡偏熊、弱熊、熊市

### 方式 D：防守模式

仅 1 次买入，不超过计划仓位 30%，严格止损。

**适用场景**：强熊

### 选择依据

根据 market-regime-classifier.md 输出的市场状态选择策略，而非手动判断。

## 分段卖出策略

### 正常卖出

- 单次最多卖出：持仓的 50%
- 分批止盈，逐步锁定收益

### 止损卖出（例外情况）

- **条件**：持仓亏损超过动态止损线
- **操作**：允许一次性全部卖出
- **原则**：止损要果断，不要分批

### 动态止损线

| 市场状态 | 止损线 |
|----------|--------|
| 强牛/牛/弱牛 | -20% |
| 震荡偏牛 | -20% |
| 震荡偏熊 | -15% |
| 弱熊 | -15% |
| 熊/强熊 | -10% |

## 买入数量计算公式

```
1. 获取可用余额 (available_amt)
2. 获取当前价格 (current_price)
3. 获取市场状态 → 确定计划仓位比例
4. 计算计划仓位金额 = min(总资产 × 动态比例, 可用资金)
5. 计算最大可买数量（整手）:
   max_qty = floor(计划仓位金额 / (price * 100)) * 100
6. 计算所需金额:
   required = max_qty * price + 5 (手续费)
7. 检查金额是否充足:
   if available_amt < required:
     放弃买入
```
