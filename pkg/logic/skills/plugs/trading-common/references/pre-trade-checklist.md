# 交易前置检查清单 (v2.4.0)

> Phase 3 风险管理增强 — 每次交易前的强制执行检查

## 检查清单

在做出任何买卖决策前，必须完成以下 4 项检查，结果写入推理链。

### 1. 市场环境检查

加载 `trading-common/references/market-regime-classifier.md` 确认当前市场状态：

| 市场状态 | 是否允许交易 | 限制 |
|----------|-------------|------|
| 强牛/牛/弱牛 | 允许 | 按动态仓位上限 |
| 震荡偏牛/偏熊 | 允许 | 仓位减半 |
| 弱熊/熊 | 谨慎 | 仅原有持仓的止盈止损 |
| 强熊 | 禁止开仓 | 仅允许卖出减仓 |
| 极端 | 禁止一切操作 | HOLD |

**输出**：`市场状态: [xxx], 交易许可: [允许/谨慎/禁止]`

### 2. 交易频率检查

调用 `get_transactions(date_from="<today>", date_to="<today>")`:
- 统计当日 BUY 类型交易次数
- 统计当日 SELL 类型交易次数

| 当日交易次数 | 处理 |
|-------------|------|
| ≤3 次 | 正常 |
| 4-5 次 | 警告: 接近过度交易上限 |
| >5 次 | 拒绝: 过度交易，今日暂停 |

**输出**：`今日交易: 买入 X 次 / 卖出 Y 次, 状态: [正常/警告/拒绝]`

### 3. 决策一致性检查

对比 session 中上次决策：
- 上次卖出 → 本次买入同一股票？→ 矛盾警告，需提供反转理由
- 上次买入 → 本次卖出同一股票？→ 检查持仓时间，< 1 天则为日内翻转警告
- 上次 HOLD → 本次要交易？→ 确认条件是否发生了变化

**输出**：`上次决策: [xxx], 一致性: [一致/矛盾], 说明: [...]`

### 4. HOLD 替代方案

强制考虑 "什么都不做" 选项：
- 如果 HOLD：无亏损风险，无交易费用，可等待更明确信号
- 如果交易：可能的风险是什么？最大的收益是什么？
- 至少列出 1 个支持 HOLD 的理由

**输出**：`HOLD 理由: [至少 1 个], 交易必要性: [必要/可选/不必要]`

## 检查结果汇总

```
□ 1. 市场环境: [状态] — [允许/谨慎/禁止]
□ 2. 交易频率: 买X/卖Y — [正常/警告/拒绝]
□ 3. 决策一致性: [一致/矛盾] — [说明]
□ 4. HOLD 考虑: [理由]
→ 最终: [继续决策 / HOLD]
```

## 使用说明

1. morning-analysis/SKILL.md Step 2 决策前加载此文件
2. afternoon-trade/SKILL.md Step 2 决策前加载此文件
3. 检查结果写入推理链的"假设"部分
4. 任一检查项返回"拒绝"时，强制输出 HOLD
