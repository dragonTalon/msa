# 决策推理链模板 - 6段式

## 概述

每个买卖决策必须用此 6 段推理链输出，确保决策逻辑完整、可追溯、可复盘。

---

## 推理链结构

```
触发条件 → 理由详述 → 反向因素 → 信息质量 → 市场状态 → 决策输出
```

---

## 第 1 段: 触发条件评估

**目的**: 明确是什么触发了这个决策考虑。

### 内容
```markdown
### 1. 触发条件

- 触发类型: [止盈/止损/技术信号/消息驱动/策略规则/定期检查]
- 触发来源: [具体描述]
- 触发强度: [强/中/弱]
- 是否满足最低触发标准: [是/否]

如果"否" → 终止推理，不进入后续段
```

### 判断标准
| 触发强度 | 条件 | 后续处理 |
|----------|------|----------|
| 强 | 策略规则触发、止损触发 | 必须执行 |
| 中 | 技术信号共振、消息明确 | 结合后续段判断 |
| 弱 | 单一技术信号、模糊消息 | 仅记录观察，不执行 |

---

## 第 2 段: 理由详述

**目的**: 详细阐述支持和反对该决策的全部理由。

### 内容
```markdown
### 2. 理由详述

#### 支持理由
1. [理由 1] - 权重: [高/中/低]
2. [理由 2] - 权重: [高/中/低]
...

#### 反对理由
1. [理由 1] - 权重: [高/中/低]
...

#### 理由对比
- 支持理由总权重: [高/中/低]
- 反对理由总权重: [高/中/低]
- 初步倾向: [执行/观望/放弃]
```

---

## 第 3 段: 反向因素

**目的**: 强制考虑反面因素，对抗确认偏误。

### 内容
```markdown
### 3. 反向因素检查

#### 如果决策是"买入":
- [ ] 是否因为看到别人赚钱而想买? (从众心理)
- [ ] 是否因为新闻标题刺激而想买? (情绪驱动)
- [ ] 是否错过了最佳买点而在追高? (FOMO)
- [ ] 如果现在空仓，还会买这个吗? (独立判断)
- [ ] 最坏情况是什么? 能承受吗?

#### 如果决策是"卖出":
- [ ] 是否因为恐慌而想卖? (恐慌抛售)
- [ ] 是否因为短期波动而动摇? (缺乏耐心)
- [ ] 卖出后如果大涨，会后悔吗? (确认是否深思熟虑)
- [ ] 最初买入的理由是否仍然成立? (逻辑一致性)
- [ ] 是否因为亏损厌恶而拖延止损? (损失厌恶)

#### 强制性反向搜索
- [ ] 已搜索反对该决策的观点/信息
- [ ] 已考虑至少 1 个不执行该决策的合理场景
```

---

## 第 4 段: 信息质量标注

**目的**: 评估支撑本决策的信息质量，直接引用 news-quality-gate.md 的输出。

### 内容
```markdown
### 4. 信息质量

引用自「信息质量门」评估:

| 维度 | 本决策涉及信息的评级 | 对决策的影响 |
|------|---------------------|-------------|
| 来源可信度 | [高/中/低] | [说明] |
| 时效性 | [新鲜/可接受/已定价] | [说明] |
| 拥挤度 | [正常/关注/拥挤] | [说明] |
| 完整性 | [充分/可接受/不足] | [说明] |

综合判定: [可决策 / 需补充 / 暂停]
```

---

## 第 5 段: 市场状态上下文

**目的**: 引用市场状态分类器的结果，说明市场环境对决策的影响。

### 内容
```markdown
### 5. 市场状态

引用自「市场状态分类器」:

- 当前状态: [STRONG_BULL / BULL / WEAK_BULL / RANGING_BULLISH / RANGING_BEARISH / WEAK_BEAR / BEAR / STRONG_BEAR]
- 趋势得分: [+/-xx]
- 波动率得分: [+/-xx]
- 情绪得分: [+/-xx]

#### 对本决策的影响
- 仓位上限: [x%] → 本决策使用 [x%]
- 追高限制: [x%] → 当前涨幅 [x%] [符合/超出]
- 决策逻辑调整: [买入放宽/买入收紧/卖出放宽/卖出收紧]
- 特别提示: [市场状态相关的特别提醒]
```

---

## 第 6 段: 决策输出

**目的**: 综合前 5 段，输出明确、可执行的决策。

### 内容
```markdown
### 6. 决策输出

- 决策类型: [买入/卖出/持有/观察]
- 目标代码: [code]
- 目标名称: [name]
- 操作数量: [qty]股
- 操作价格: [价格]元 (当前价)
- 决策置信度: [高/中/低]

#### 决策逻辑总结
基于 [触发条件] → [核心理由] → 考虑 [反向因素] →
信息质量 [评级] → 市场状态 [状态] → 做出 [决策]

#### 预期结果
- 预期收益: [x%]
- 预期持仓时间: [x 天/周]
- 止损位: [价格] (-[x%])
- 止盈位: [价格] (+[x%])
```

---

## 完整推理链示例

```markdown
## 决策推理链: 买入 贵州茅台 (sh600519)

### 1. 触发条件
- 触发类型: 技术信号 + 策略规则
- 触发来源: 早盘回调至 1800 元支撑位 + MACD 金叉
- 触发强度: 中
- 是否满足最低触发标准: 是

### 2. 理由详述
#### 支持理由
1. 回调至关键支撑位 1800 元企稳 - 权重: 高
2. MACD 日线金叉形成 - 权重: 中
3. 北向资金近期持续流入白酒板块 - 权重: 中

#### 反对理由
1. 白酒板块整体估值仍偏高 - 权重: 中
2. 大盘短期方向不明 - 权重: 低

#### 理由对比
- 支持理由总权重: 高
- 反对理由总权重: 中
- 初步倾向: 执行

### 3. 反向因素检查
- [x] 不是因为看到别人赚钱而想买
- [x] 不是受新闻标题刺激
- [x] 如果空仓仍会考虑买入
- [x] 最坏情况: 跌破 1750 止损, 亏损约 2.8%

### 4. 信息质量
| 维度 | 评级 | 对决策的影响 |
|------|------|-------------|
| 来源可信度 | 高 (S:1 A:3 B:1) | 技术数据可靠 |
| 时效性 | 新鲜 | 实时行情 |
| 拥挤度 | 正常 (板块涨幅 0.8%) | 无追高顾虑 |
| 完整性 | 充分 | 多维度信息齐备 |
综合判定: 可决策

### 5. 市场状态
- 当前状态: WEAK_BULL
- 仓位上限: 30% → 本决策使用 15%
- 追高限制: 3% → 当前涨幅 0.5%, 符合
- 决策逻辑调整: 买入条件轻度放宽

### 6. 决策输出
- 决策类型: 买入
- 目标代码: sh600519
- 目标名称: 贵州茅台
- 操作数量: 100股
- 操作价格: 1810.00元
- 决策置信度: 中

#### 决策逻辑总结
基于 技术信号触发 → 支撑位企稳+MACD金叉 → 考虑了估值偏高但支撑有效 →
信息质量高 → 市场弱牛 → 买入 100 股试探性建仓

#### 预期结果
- 预期收益: +5%
- 预期持仓时间: 5 个交易日
- 止损位: 1750.00 (-3.3%)
- 止盈位: 1900.00 (+5.0%)
```

---

## 使用规则

1. **每个决策必须输出完整 6 段**，不可跳过任何一段
2. **第 1 段判定"否"则终止**，不浪费时间展开后续推理
3. **第 4 段直接引用** news-quality-gate.md 的输出
4. **第 5 段直接引用** market-regime-classifier.md 的输出
5. **第 6 段必须可执行**：明确代码、数量、价格、止损止盈
