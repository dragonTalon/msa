# 提案：session-resume

实现会话持久化与恢复功能，支持用户保存对话历史并在后续恢复继续对话。

## 知识上下文

### 相关历史问题

知识库中未找到与会话管理直接相关的问题。此为新功能领域。

### 需避免的反模式

未识别到特定反模式。但需注意：
- 文件写入时应避免频繁打开关闭，推荐使用追加写入
- 并发写入需要考虑文件锁

### 推荐模式

✅ **推荐**：考虑使用以下模式：

- **browser-manager-singleton**：单例模式管理全局资源，可参考用于 SessionManager 设计
- **error-handling-strategy**：错误分类处理，文件操作失败时提供清晰错误信息

## 为什么

当前 MSA 缺少会话持久化能力：

1. **TUI 模式**：对话仅在内存中，退出后丢失
2. **CLI 模式**：单轮问答无法追溯历史
3. **无法恢复**：无法继续之前的对话上下文

用户需要：
- 保存有价值的对话内容
- 在中断后继续之前的工作
- 跨会话保持对话连贯性

## 变更内容

- 新增 `pkg/session/` 模块，管理会话生命周期
- TUI 模式启动时创建会话，记录用户问题与助手回复
- CLI 模式 (`-q`) 执行时记录单轮问答
- 新增 `--resume` 参数恢复历史会话
- 会话文件存储在 `~/.msa/memory/YYYY/MM/DD/{uuid}.md`

## 能力

### 新增能力

- `session-persist`：会话持久化能力，自动保存对话历史到 Markdown 文件
- `session-resume`：会话恢复能力，通过 `--resume` 参数加载历史会话继续对话

### 修改能力

无

## 非目标

- 不实现会话列表/选择功能
- 不实现会话数量限制或自动清理
- 不实现敏感信息加密
- 不实现会话导出/导入功能

- TUI 模式：欢迎信息显示当前会话 ID
