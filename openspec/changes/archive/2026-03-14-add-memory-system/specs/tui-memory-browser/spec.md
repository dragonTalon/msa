# 规格：tui-memory-browser

> 📚 **知识上下文**：基于 Bubble Tea TUI 框架和现有的 selector 组件模式

## ADDED Requirements

### 需求：记忆主页导航

系统必须提供一个主页作为记忆浏览器的入口点。

#### 场景：进入记忆主页

- **当** 用户在聊天界面输入 `/remember`
- **那么** 切换到记忆主页视图
- **并且** 显示 4 个主要选项：
  - 📖 历史会话
  - 💡 知识库
  - 🔍 搜索记忆
  - 📊 统计信息
- **并且** 每个选项显示对应的数量（如果有）
- **并且** 支持 1-4 数字快捷键

#### 场景：从记忆主页返回聊天

- **当** 用户按 ESC 或 q
- **那么** 返回到聊天界面
- **并且** 聊天状态保持不变（未丢失输入）

### 需求：会话列表视图

系统必须提供可浏览、可搜索的会话列表。

#### 场景：显示会话列表

- **当** 用户从主页选择"历史会话"
- **那么** 显示所有历史会话，按最新优先
- **并且** 每个会话显示：
  - 日期和时间
  - 消息数量
  - 简短摘要（用户第一条输入）
  - 标签
  - 提及的股票代码
- **并且** 支持分页（默认 10 条/页）

#### 场景：会话列表搜索

- **当** 用户在会话列表中按 `/` 进入搜索模式
- **那么** 显示搜索提示符
- **并且** 实时过滤显示匹配的会话
- **并且** 支持按日期、标签、股票代码过滤
- **并且** 按 ESC 退出搜索模式

#### 场景：从会话列表查看详情

- **当** 用户按 Enter 选择一个会话
- **那么** 切换到会话详情视图
- **并且** 传递会话 ID 用于加载完整内容

#### 场景：从会话列表删除会话

- **当** 用户按 d 删除选中的会话
- **那么** 显示确认提示
- **并且** 确认后删除会话文件
- **并且** 更新索引并刷新列表

### 需求：会话详情视图

系统必须支持查看完整的历史会话内容。

#### 场景：加载会话详情

- **当** 进入会话详情视图
- **那么** 加载完整的会话 JSON 文件
- **并且** 显示会话元数据：
  - 开始和结束时间
  - 会话时长
  - 使用的模型
  - 提及的股票
- **并且** 显示完整的对话历史
- **并且** 支持滚动浏览

#### 场景：会话详情搜索

- **当** 用户在会话详情中按 `/` 搜索
- **那么** 跳转到第一个匹配的消息
- **并且** 高亮显示匹配内容
- **并且** 支持 n/N 在匹配项之间导航
- **并且** 按 ESC 退出搜索

#### 场景：会话详情滚动

- **当** 用户使用方向键或 PgUp/PgDn
- **那么** 平滑滚动消息列表
- **并且** 显示当前位置（如 5/23）
- **并且** 显示滚动指示器
- **并且** 支持 g 跳到开头，G 跳到结尾

#### 场景：会话详情导出

- **当** 用户按 e 导出会话
- **那么** 生成格式化的文本文件
- **并且** 保存到用户指定的路径
- **并且** 显示成功/失败提示

### 需求：知识列表视图

系统必须提供按类型浏览知识的能力。

#### 场景：显示知识类型选择

- **当** 用户从主页选择"知识库"
- **那么** 显示 6 个知识类型：
  - 👤 用户画像
  - 📊 关注列表
  - 📚 学过的概念
  - 📋 交易策略
  - ❓ 常见问答
  - 🔍 其他洞察
- **并且** 每个类型显示数量
- **并且** 支持 1-6 数字快捷键

#### 场景：显示特定类型的知识

- **当** 用户选择某个知识类型
- **那么** 显示该类型的所有知识
- **并且** 每个知识显示：
  - 名称/标题
  - 描述/摘要
  - 提取时间
  - 来源会话
- **并且** 支持分页和过滤

#### 场景：查看知识详情

- **当** 用户按 Enter 选择一个知识
- **那么** 显示完整的知识内容
- **并且** 显示来源会话的链接
- **并且** 如果是概念，显示理解程度

### 需求：搜索视图

系统必须提供统一的搜索界面。

#### 场景：进入搜索视图

- **当** 用户从主页选择"搜索记忆"
- **那么** 显示搜索输入框
- **并且** 显示过滤器选项（类型、日期范围）
- **并且** 显示搜索历史

#### 场景：执行搜索

- **当** 用户输入搜索词并按 Enter
- **那么** 显示搜索结果
- **并且** 结果包含会话和知识
- **并且** 按相关性排序
- **并且** 每个结果显示匹配片段

#### 场景：打开搜索结果

- **当** 用户按 Enter 选择搜索结果
- **那么** 如果是会话，跳转到会话详情并高亮匹配
- **并且** 如果是知识，跳转到知识详情
- **并且** 在详情中显示匹配内容

### 需求：统计视图

系统必须提供使用统计和趋势分析。

#### 场景：显示统计概览

- **当** 用户从主页选择"统计信息"
- **那么** 显示：
  - 总会话数
  - 总消息数
  - 总知识数
  - 最早和最新会话时间
  - 最常查询的股票 TOP 5
  - 学得最多的概念 TOP 5
- **并且** 使用图表或表格可视化

#### 场景：显示使用趋势

- **当** 用户查看统计详情
- **那么** 按月显示会话数量趋势
- **并且** 按周显示活跃度
- **并且** 显示知识增长曲线

### 需求：键盘一致性

所有记忆浏览器视图必须使用一致的键盘快捷键。

#### 场景：通用导航键

- **当** 用户在任何记忆视图
- **那么** 以下快捷键应保持一致：
  - `ESC` - 返回上一级或退出
  - `q` - 退出记忆浏览器（返回聊天）
  - `↑/↓` 或 `j/k` - 上下移动
  - `Enter` - 确认选择
  - `/` - 进入搜索模式
  - `PgUp/PgDn` - 翻页

#### 场景：上下文帮助

- **当** 用户在任何视图
- **那么** 底部显示可用的快捷键
- **并且** 按 `?` 显示详细帮助
- **并且** 帮助包含所有快捷键和功能说明

### 需求：视图状态管理

系统必须正确管理视图状态和转换。

#### 场景：视图栈管理

- **当** 用户在视图间导航
- **那么** 维护视图栈（聊天 → 主页 → 列表 → 详情）
- **并且** ESC 返回上一级视图
- **并且** 保留滚动位置和过滤状态
- **并且** 正确处理视图生命周期

#### 场景：视图数据刷新

- **当** 用户返回某个视图
- **那么** 如果数据可能已更新，刷新显示
- **并且** 会话列表应在退出详情后刷新（可能被删除）
- **并且** 知识缓存应在提取新知识后更新

> 📚 **推荐模式**：参考现有的 model_selector.go 实现
> 复用选择器组件和样式，保持 UI 一致性

### 需求：错误处理和恢复

系统必须优雅地处理错误情况。

#### 场景：文件读取失败

- **当** 尝试读取会话或知识文件失败
- **那么** 显示友好的错误消息
- **并且** 提供重试选项
- **并且** 不影响其他功能

#### 场景：索引损坏

- **当** 索引文件损坏或不存在
- **那么** 显示"正在重建索引..."
- **并且** 自动扫描文件重建
- **并且** 完成后刷新视图

> 📚 **推荐模式**：使用错误处理策略（pattern 003）
> UI 层面的错误不应导致程序崩溃，应提供清晰的反馈和恢复路径
