# 规格：memory-search

> 📚 **知识上下文**：基于模式 003（错误处理策略）- 搜索性能优化

## ADDED Requirements

### 需求：跨会话关键词搜索

系统必须支持在所有历史会话和知识中搜索关键词。

#### 场景：简单关键词搜索

- **当** 用户在搜索视图中输入关键词
- **那么** 同时搜索会话和知识
- **并且** 返回匹配结果列表，按相关性排序
- **并且** 每个结果显示：
  - 类型标签（会话/知识）
  - 标题/摘要
  - 匹配内容片段（高亮关键词）
  - 时间戳
- **并且** 搜索应在 500ms 内完成（使用索引）

#### 场景：多关键词搜索

- **当** 用户输入多个关键词（空格分隔）
- **那么** 搜索包含所有关键词的结果（AND 逻辑）
- **并且** 支持引号包裹精确匹配（如 "腾讯 股价"）
- **并且** 优先显示完全匹配的结果

### 需求：股票代码搜索

系统必须支持通过股票代码快速查找相关会话。

#### 场景：精确股票代码搜索

- **当** 用户搜索股票代码（如 "00700.HK"）
- **那么** 显示所有提及该股票的会话
- **并且** 显示相关的关注列表知识
- **并且** 搜索不区分大小写

#### 场景：股票名称搜索

- **当** 用户搜索股票名称（如 "腾讯"）
- **那么** 查找包含该名称的会话
- **并且** 尝试匹配关联的股票代码
- **并且** 同时返回代码和名称匹配的结果

### 需求：时间范围过滤

用户应能够按时间范围缩小搜索结果。

#### 场景：指定日期范围

- **当** 用户设置开始和结束日期
- **那么** 只搜索该时间范围内的会话和知识
- **并且** 日期支持相对格式：
  - `today`, `yesterday`
  - `7d ago`, `30d ago`
  - `2024-03-01`

#### 场景：按月浏览

- **当** 用户选择特定月份
- **那么** 显示该月的所有会话
- **并且** 按日期分组显示
- **并且** 显示该月的统计信息（会话数、消息数）

### 需求：搜索结果分页和导航

搜索结果必须支持分页和键盘导航。

#### 场景：分页显示结果

- **当** 搜索结果超过一页（默认 10 条）
- **那么** 显示当前页码和总页数
- **并且** 支持 PgUp/PgDn 翻页
- **并且** 显示滚动指示器（上方/下方还有 N 条）

#### 场景：打开搜索结果

- **当** 用户按 Enter 选择一个搜索结果
- **那么** 如果是会话，跳转到会话详情
- **并且** 如果是知识，跳转到知识详情
- **并且** 在详情中高亮显示匹配内容

### 需求：搜索历史和建议

系统应记录搜索历史并提供智能建议。

#### 场景：显示搜索历史

- **当** 用户进入搜索视图
- **那么** 显示最近的 5 个搜索词
- **并且** 支持方向键快速选择历史搜索
- **并且** 按 Enter 重新执行历史搜索

#### 场景：搜索建议

- **当** 用户输入搜索关键词
- **那么** 根据索引提供自动完成建议
- **并且** 建议包括：
  - 历史搜索词
  - 常见股票代码
  - 常见概念名称
- **并且** 按 Tab 接受建议

### 需求：高级搜索过滤器

系统应支持组合多个搜索条件。

#### 场景：按类型过滤

- **当** 用户选择搜索类型（会话/知识/全部）
- **那么** 只在选定类型中搜索
- **并且** 类型选择应持久化（下次使用记住偏好）

#### 场景：按标签过滤

- **当** 用户选择特定标签（如 "技术指标"）
- **那么** 只显示带有该标签的会话
- **并且** 显示可用的标签列表及其数量

#### 场景：组合过滤

- **当** 用户同时设置多个过滤条件
- **那么** 应用所有过滤条件（AND 逻辑）
- **并且** 清晰显示当前活跃的过滤器
- **并且** 支持快速清除单个或全部过滤器

### 需求：搜索性能优化

搜索操作必须使用索引优化，避免全文件扫描。

#### 场景：使用内存索引

- **当** 执行搜索操作时
- **那么** 使用内存索引进行查找
- **并且** 不扫描实际文件内容（除了加载详情时）
- **并且** 索引包含：关键词、股票代码、标签、摘要

#### 场景：索引失效重建

- **当** 索引文件损坏或不存在
- **那么** 自动扫描会话和知识文件重建索引
- **并且** 显示"正在重建索引..."提示
- **并且** 重建完成后自动执行搜索

> 📚 **推荐模式**：使用错误处理策略（pattern 003）
> 搜索失败时回退到基础实现，不应导致整个功能不可用

### 需求：搜索结果导出

用户应能够导出搜索结果。

#### 场景：导出为文本

- **当** 用户选择导出搜索结果
- **那么** 生成格式化的文本报告
- **并且** 包含：搜索条件、结果摘要、完整内容
- **并且** 保存到用户指定的路径

#### 场景：导出为 JSON

- **当** 用户选择导出为 JSON 格式
- **那么** 生成机器可读的 JSON 文件
- **并且** 包含所有搜索结果的完整数据
- **并且** 支持后续导入或分析
