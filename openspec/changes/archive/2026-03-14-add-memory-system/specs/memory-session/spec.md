# 规格：memory-session

> 📚 **知识上下文**：基于模式 004（单例模式）和 003（错误处理策略）

## ADDED Requirements

### 需求：会话持久化

系统必须在每次聊天会话结束时自动保存完整的对话记录，以便后续检索和分析。

#### 场景：正常退出时保存会话

- **当** 用户通过 Ctrl+C 或 ESC 正常退出聊天
- **那么** 系统应将完整会话保存到 `~/.msa/remember/sessions/<YYYY-MM>/<session-id>.json`
- **并且** 会话文件应包含所有消息（用户和助手）、元数据、上下文信息
- **并且** 保存操作不应阻塞用户退出

#### 场景：异常中断时保存会话

- **当** 用户强制终止进程（如 SIGKILL）
- **那么** 系统应尽力保存已记录的消息
- **并且** 下次启动时应检测并恢复未完整保存的会话

> 📚 **历史问题参考**：基于缓解策略 - 存储文件损坏可能导致记忆丢失
> 使用 JSON 格式便于恢复，实施写入前验证

### 需求：会话元数据记录

每个保存的会话必须包含结构化的元数据，便于快速检索和统计。

#### 场景：记录会话基本元数据

- **当** 保存会话时
- **那么** 必须记录以下元数据：
  - 会话 ID（UUID）
  - 开始时间和结束时间
  - 消息总数
  - 使用的 LLM 模型
  - 提及的股票代码列表
  - 自动生成的标签

#### 场景：提取会话标签

- **当** 分析会话内容时
- **那么** 系统应自动生成描述性标签：
  - "股票查询" - 包含股票查询行为
  - "技术指标" - 讨论了技术分析指标
  - "交易操作" - 包含买入/卖出指令
  - "账户管理" - 涉及账户操作

### 需求：会话检索和列表

系统必须提供按时间顺序浏览历史会话的能力。

#### 场景：按时间倒序列出会话

- **当** 用户执行 `/remember` 进入会话列表
- **那么** 显示所有历史会话，按最新优先排序
- **并且** 每个会话显示：日期、消息数、简短摘要、标签
- **并且** 支持分页显示（默认每页 10 条）

#### 场景：按日期过滤会话

- **当** 用户指定日期范围或特定月份
- **那么** 只显示该时间范围内的会话
- **并且** 过滤操作应在 100ms 内完成（使用索引）

#### 场景：按股票代码过滤会话

- **当** 用户搜索特定股票代码（如 "00700.HK"）
- **那么** 显示所有提及该股票的会话
- **并且** 股票代码支持模糊匹配（如 "腾讯" 也能找到 00700.HK）

### 需求：会话详情查看

系统必须支持查看完整的历史会话内容。

#### 场景：加载完整会话详情

- **当** 用户选择查看某个会话
- **那么** 显示完整的对话历史
- **并且** 包含会话元数据（时间、模型、股票等）
- **并且** 高亮显示提取的知识点
- **并且** 支持滚动浏览（使用 TUI 分页）

#### 场景：会话内搜索

- **当** 用户在会话详情中输入搜索关键词
- **那么** 跳转到第一个匹配的消息
- **并且** 高亮显示匹配内容
- **并且** 支持 n/N 键在匹配项之间跳转

### 需求：会话删除

用户应能够删除不需要的历史会话。

#### 场景：删除单个会话

- **当** 用户在会话列表或详情中按 d 键
- **那么** 系统应确认删除操作
- **并且** 删除会话文件和索引中的记录
- **并且** 关联的知识文件保留（知识独立于会话）

#### 场景：批量删除旧会话

- **当** 用户执行清理命令（如 `/remember prune --older-than 30d`）
- **那么** 删除指定时间之前的所有会话
- **并且** 保留统计和摘要数据

> 📚 **推荐模式**：使用单例模式（pattern 004）管理 MemoryManager
> 确保全局只有一个存储实例，避免并发写入冲突

### 需求：存储索引

系统必须维护内存索引以加速会话检索。

#### 场景：增量更新索引

- **当** 保存新会话或删除会话时
- **那么** 立即更新内存索引
- **并且** 异步持久化索引到 `~/.msa/remember/indexes/all.json`
- **并且** 索引更新不应阻塞主流程

#### 场景：启动时加载索引

- **当** MSA 启动时
- **那么** 从 `~/.msa/remember/indexes/all.json` 加载索引
- **并且** 如果索引文件不存在或损坏，扫描会话文件重建索引
- **并且** 索引加载应在 200ms 内完成

> 📚 **推荐模式**：使用错误处理策略（pattern 003）
> 索引加载失败时回退到扫描重建，不应导致启动失败
