# 规格：chat-session

> 📚 **知识上下文**：基于模式 004（单例模式）- MemoryManager 应使用单例模式

## MODIFIED Requirements

### 需求：会话初始化增强

聊天会话初始化时必须同时初始化记忆管理器。

#### 场景：启动时初始化记忆系统

- **当** 创建新的 Chat 实例
- **那么** 初始化 MemoryManager（单例）
- **并且** 生成唯一的会话 ID（UUID）
- **并且** 记录会话开始时间
- **并且** 创建空的短期内存（最多 10 条消息）
- **并且** 如果记忆系统初始化失败，记录警告但不阻止聊天

> 📚 **推荐模式**：使用单例模式（pattern 004）
> MemoryManager 应使用 sync.Once 确保全局唯一实例

### 需求：消息记录增强

每条消息必须同时记录到历史和记忆系统。

#### 场景：用户输入消息

- **当** 用户输入消息并按 Enter
- **那么** 将消息添加到 Chat.history（现有行为）
- **并且** 调用 memoryManager.AddMessage() 记录到记忆
- **并且** 更新短期内存（保持最近 10 条）
- **并且** 持久化短期内存到 `~/.msa/remember/short-term/current.json`
- **并且** 如果记录失败，不影响聊天继续进行

#### 场景：AI 回复消息

- **当** AI 生成回复消息
- **那么** 将消息添加到 Chat.history（现有行为）
- **并且** 调用 memoryManager.AddMessage() 记录到记忆
- **并且** 更新会话上下文（如提取股票代码）

#### 场景：更新会话上下文

- **当** 记录消息到记忆系统
- **那么** 分析消息内容更新会话上下文：
  - 提取股票代码（如 00700.HK、600519）
  - 检测使用的工具
  - 检测交易操作
  - 生成标签候选
- **并且** 上下文信息用于后续的知识提取

### 需求：会话结束增强

会话结束时必须保存完整会话并触发知识提取。

#### 场景：正常退出保存会话

- **当** 用户按 Ctrl+C 或 ESC 退出
- **那么** 调用 memoryManager.EndSession()
- **并且** 保存完整会话到 `~/.msa/remember/sessions/<YYYY-MM>/<session-id>.json`
- **并且** 异步触发 AI 知识提取
- **并且** 更新会话索引
- **并且** 清空短期内存文件
- **并且** 退出不应被保存操作阻塞

#### 场景：异常中断恢复

- **当** 检测到未完整保存的会话（如短期内存文件存在但会话不存在）
- **那么** 下次启动时提示用户
- **并且** 询问是否恢复会话
- **并且** 如果用户确认，将短期内存内容合并到新会话

> 📚 **推荐模式**：遵循 AI 助手工作流程（pattern 002）
> 知识提取是异步任务，不应阻塞用户退出或自动执行影响性操作

### 需求：短期内存管理

系统必须维护最近 10 条消息的快速访问内存。

#### 场景：维护短期内存

- **当** 添加新消息时
- **那么** 将消息添加到短期内存
- **并且** 如果超过 10 条，移除最旧的
- **并且** 只保留用户和助手的消息（过滤系统消息）
- **并且** 持久化到 `short-term/current.json`

#### 场景：启动时恢复短期内存

- **当** MSA 启动时
- **那么** 检查是否存在短期内存文件
- **并且** 如果存在，询问用户是否继续上次会话
- **并且** 如果用户确认，加载短期内存内容到新会话
- **并且** 如果用户拒绝，删除短期内存文件

### 需求：记忆系统状态显示

用户应能看到记忆系统的状态。

#### 场景：显示记忆系统状态

- **当** 用户输入 `/status` 或类似命令
- **那么** 显示记忆系统状态：
  - 当前会话 ID
  - 会话时长
  - 消息数量
  - 记忆系统状态（启用/禁用）
  - 知识库大小
- **并且** 如果记忆系统被禁用，显示启用方法

#### 场景：记忆系统错误通知

- **当** 记忆系统操作失败
- **那么** 在下次发送消息时显示系统提示
- **并且** 说明错误类型（如"保存会话失败"）
- **并且** 不影响聊天继续进行
- **并且** 只通知一次（不重复）

### 需求：记忆系统可选禁用

用户应能选择不使用记忆系统。

#### 场景：禁用记忆系统

- **当** 用户设置环境变量 `MSA_MEMORY_ENABLED=false`
- **那么** 跳过记忆系统初始化
- **并且** Chat 行为与现有版本完全一致
- **并且** 不显示任何记忆相关提示

#### 场景：运行时禁用记忆

- **当** 用户执行 `/remember disable` 命令
- **那么** 停止记录消息到记忆
- **并且** 会话结束时不再保存
- **并且** 设置应持久化（下次启动仍然禁用）
- **并且** 可以通过 `/remember enable` 重新启用

---

**Reason for MODIFIED Requirements**：
记忆系统是对现有会话管理能力的增强，不是全新功能。通过修改现有能力，确保向后兼容性和平滑集成。

**Migration Path**：
1. 现有代码无需修改即可工作（记忆系统是可选的）
2. 启用记忆系统后，会话自动被记录和保存
3. 用户可以随时禁用记忆系统，恢复原有行为
