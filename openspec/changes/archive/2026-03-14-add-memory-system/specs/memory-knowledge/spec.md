# 规格：memory-knowledge

> 📚 **知识上下文**：基于模式 002（AI 助手工作流程）和风险评估（敏感信息脱敏）

## ADDED Requirements

### 需求：知识自动提取

系统必须在每次会话结束时使用 AI 从对话中提取结构化知识。

#### 场景：会话结束触发知识提取

- **当** 用户退出聊天会话
- **那么** 异步调用 AI 分析对话内容
- **并且** 提取以下类型的知识：
  - 用户画像（投资风格、风险偏好）
  - 关注列表（股票、板块）
  - 学过的概念（指标、术语）
  - 交易策略
  - 常见问答
- **并且** 提取过程不阻塞用户退出

#### 场景：知识提取失败处理

- **当** AI 提取失败（网络错误、超时等）
- **那么** 记录错误日志
- **并且** 会话仍然正常保存
- **并且** 不影响下次使用

> 📚 **推荐模式**：遵循 AI 助手工作流程（pattern 002）
> 知识提取是异步后台任务，不应阻塞用户操作或自动执行影响性操作

### 需求：知识结构化存储

提取的知识必须按类型分目录存储，便于检索和聚合。

#### 场景：保存知识到对应目录

- **当** AI 提取出一个知识对象
- **那么** 根据知识类型保存到对应目录：
  - 用户画像 → `knowledge/profile/`
  - 关注列表 → `knowledge/watchlist/`
  - 学过的概念 → `knowledge/concepts/`
  - 交易策略 → `knowledge/strategies/`
  - 常见问答 → `knowledge/qanda/`
  - 其他洞察 → `knowledge/insights/`
- **并且** 每个知识文件使用 UUID 命名
- **并且** 文件内容为 JSON 格式

#### 场景：知识关联会话

- **当** 保存知识文件时
- **那么** 记录来源会话 ID
- **并且** 在会话索引中记录关联的知识 ID
- **并且** 支持从会话详情跳转到相关知识

### 需求：敏感信息过滤

在保存对话和提取知识前，必须过滤敏感信息。

#### 场景：检测 API Key

- **当** 分析消息内容时
- **那么** 检测常见 API Key 模式：
  - `sk-` 开头的字符串（OpenAI/SiliconFlow）
  - `Bearer ` 后的 token
  - 其他已知 API Key 格式
- **并且** 将检测到的 API Key 替换为 `***REDACTED***`

#### 场景：检测密码

- **当** 消息包含密码相关内容时
- **那么** 检测常见密码指示词：
  - "密码:"
  - "password:"
  - "passwd:"
- **并且** 将密码值替换为 `***REDACTED***`

> 📚 **风险评估参考**：用户敏感信息可能被记录
> 必须在保存前实施内容过滤，检测并脱敏敏感信息

### 需求：知识聚合和查询

系统必须支持聚合多个历史知识，提供统一的用户画像。

#### 场景：聚合用户画像

- **当** 查询用户画像时
- **那么** 合并所有历史画像知识
- **并且** 取最新的投资风格和偏好
- **并且** 聚合关注列表，去重股票代码
- **并且** 收集所有学过的概念

#### 场景：查询特定类型知识

- **当** 用户浏览知识库时
- **那么** 按类型分组显示知识
- **并且** 每个知识显示：
  - 名称/标题
  - 描述/摘要
  - 提取时间
  - 来源会话（可点击跳转）

### 需求：知识质量评估

AI 提取的知识应包含质量评估信息。

#### 场景：评估概念理解程度

- **当** 提取"学过的概念"知识时
- **那么** AI 应评估用户的理解程度（0.0-1.0）：
  - 0.9-1.0: 用户表现出深入理解
  - 0.7-0.9: 用户基本理解，能应用
  - 0.5-0.7: 用户初步了解
  - 0.3-0.5: 用户听过但不熟悉
  - 0.0-0.3: 用户完全陌生
- **并且** 将评估结果存储在 `confidence` 字段

#### 场景：检测知识冲突

- **当** 新提取的知识与历史知识冲突时
- **那么** 创建新的知识对象（不覆盖旧的）
- **并且** 记录冲突标记和时间戳
- **并且** 允许用户查看知识演变历史

### 需求：知识索引和搜索

系统必须维护知识索引以支持快速检索。

#### 场景：构建知识索引

- **当** 保存新知识时
- **那么** 更新知识索引
- **并且** 索引包含：知识 ID、类型、摘要、时间、关键词
- **并且** 支持按类型、时间、关键词快速过滤

#### 场景：搜索知识

- **当** 用户在知识视图中搜索
- **那么** 支持以下搜索方式：
  - 关键词搜索（名称、描述）
  - 类型过滤
  - 时间范围过滤
  - 股票代码过滤（针对关注列表）
- **并且** 搜索结果按相关性排序

> 📚 **推荐模式**：使用单例模式（pattern 004）
> MemoryManager 维护知识缓存，避免重复文件读取

### 需求：知识验证

提取的知识应支持用户查看和验证。

#### 场景：查看知识来源

- **当** 用户查看一个知识对象
- **那么** 显示完整的知识内容
- **并且** 显示来源会话 ID 和时间
- **并且** 提供跳转到原始会话的链接
- **并且** 如果是 AI 提取的，标注"AI 生成"

#### 场景：用户删除知识

- **当** 用户手动删除某个知识
- **那么** 删除知识文件
- **并且** 更新索引
- **并且** 保留来源会话（会话独立于知识）
- **并且** 记录删除操作到日志
