# 提案：add-memory-system

为 MSA 添加完整的对话记忆系统，支持跨会话上下文、历史回顾、AI 增强和审计功能。

## 知识上下文

### 相关历史问题

知识库中未找到直接相关的历史问题。记忆系统是新功能领域。

### 需避免的反模式

基于现有知识库的模式识别：

- **003**：不安全的 disable-web-security 标志
  - **问题所在**：为功能便利而牺牲安全性
  - **影响**：记忆系统存储用户对话，需注意敏感信息脱敏

- **001**：AI 助手不应自动执行 git commit
  - **问题所在**：自动化操作未经用户确认可能导致意外
  - **影响**：知识提取是异步操作，需确保用户知晓

### 推荐模式

✅ **推荐**：考虑使用以下模式：

- **004**：浏览器管理器单例模式
  - **描述**：使用单例模式管理全局资源
  - **使用时机**：MemoryManager 应使用单例模式管理存储和缓存

- **003**：错误处理和重试策略
  - **描述**：区分可恢复和不可恢复错误，实施适当的重试策略
  - **使用时机**：文件存储操作需要健壮的错误处理

- **002**：AI 助手工作流程指南
  - **描述**：明确 AI 助手的职责边界，不越权操作
  - **使用时机**：知识提取异步任务应遵循工作流程规范

### 风险评估

基于新功能特性，此变更具有以下风险因素：

- **中等**：存储文件损坏可能导致记忆丢失
  - **缓解措施**：实施文件写入前的验证，使用 JSON 格式便于恢复

- **中等**：大量历史会话可能影响启动性能
  - **缓解措施**：使用索引加速查询，按需加载完整会话

- **高**：用户敏感信息（如 API Key）可能被记录
  - **缓解措施**：实施内容过滤，检测并脱敏敏感信息

### 缓解策略

1. **单例管理器**：MemoryManager 使用单例模式，确保全局唯一实例
2. **异步知识提取**：会话结束时异步调用 AI，不阻塞用户退出
3. **索引优化**：构建内存索引加速搜索，避免全文件扫描
4. **隐私保护**：在保存前过滤敏感关键词（如 API Key、密码等）
5. **增量更新**：索引采用增量更新策略，避免重建开销

## 为什么

当前 MSA 每次启动都是全新会话，AI 无法"记住"之前的交互。这导致：

1. **上下文丢失**：用户需要重复说明偏好（如投资风格、关注股票）
2. **知识无法积累**：用户学过的概念（如 MACD）无法被 AI 记住
3. **历史无法追溯**：无法查看之前的对话记录和分析结果
4. **个性化缺失**：AI 无法基于用户历史提供个性化建议

## 变更内容

### 新增组件

- **pkg/logic/memory/** - 记忆系统核心逻辑
  - `store.go` - 存储抽象层和 FileStore 实现
  - `manager.go` - 记忆管理器（单例）
  - `extraction.go` - AI 知识提取

- **pkg/tui/memory_*.go** - TUI 视图组件
  - `memory_home.go` - 记忆主页
  - `memory_session_list.go` - 会话列表
  - `memory_session_detail.go` - 会话详情
  - `memory_knowledge_list.go` - 知识列表
  - `memory_search.go` - 搜索视图

- **pkg/model/memory.go** - 记忆相关数据模型

- **pkg/logic/command/memory_cmd.go** - `/remember` 命令

### 修改组件

- **pkg/tui/chat.go** - 集成记忆管理器
  - 初始化时启动会话
  - 每条消息记录到短期记忆
  - 退出时保存完整会话并提取知识

- **pkg/logic/agent/chat.go** - 注入知识到 AI Prompt
  - 在 `BuildQueryMessages` 中添加用户知识上下文

### 存储结构

```
~/.msa/remember/
├── sessions/          # 按年月组织的完整会话
├── knowledge/         # 按类型组织的提取知识
├── short-term/        # 当前会话的最近消息
└── indexes/           # 搜索索引
```

## 能力

### 新增能力

- `memory-session`：完整的会话记录和检索
  - 按时间浏览历史会话
  - 查看完整对话内容
  - 按日期、标签、股票代码过滤

- `memory-knowledge`：结构化知识提取和存储
  - 用户画像（投资风格、风险偏好）
  - 关注列表（股票、板块）
  - 学过的概念（技术指标、术语）
  - 交易策略
  - 常见问答

- `memory-search`：跨会话搜索
  - 关键词搜索
  - 股票代码搜索
  - 日期范围过滤

- `ai-context-enhancement`：AI 上下文增强
  - 启动时加载用户知识
  - 在 Prompt 中注入相关上下文
  - 提供个性化回复

- `tui-memory-browser`：TUI 记忆浏览器
  - `/remember` 命令进入
  - 多层级导航（主页 → 列表 → 详情）
  - 支持搜索和过滤

### 修改能力

- `chat-session`：会话管理能力增强
  - 新增：会话持久化
  - 新增：短期内存管理（最近10条）
  - 新增：会话结束时知识提取

## 影响

### 代码影响

- **新增文件**：约 15-20 个新文件
- **修改文件**：2-3 个核心文件（chat.go, agent 相关）
- **依赖**：无新增外部依赖

### API 影响

- 无公开 API 变更

### 性能影响

- **启动时间**：+50-200ms（加载知识缓存和索引）
- **内存占用**：+5-20MB（取决于知识库大小）
- **磁盘占用**：每个会话约 10-50KB，知识条目约 1-5KB

### 用户体验影响

- **正面**：AI 能记住用户偏好，提供更个性化的回复
- **正面**：可查看历史对话，便于回顾
- **中性**：首次退出时会触发知识提取（异步，无感知）

### 向后兼容性

- 完全兼容，记忆系统是可选功能
- 无记忆时行为与现有版本一致

---

**非目标**

- 不支持多用户记忆（单用户设计）
- 不支持云端同步（仅本地存储）
- 不支持记忆导入/导出（可在后续版本添加）
