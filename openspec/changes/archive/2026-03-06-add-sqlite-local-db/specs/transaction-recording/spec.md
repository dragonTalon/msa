# 规格：transaction-recording

交易记录管理，包括买卖记录、状态流转、部分成交处理。

## 增加需求

### 需求：交易记录创建

系统必须支持创建新的交易记录。

#### 场景：创建买入交易
- **当** 用户提交买入订单
- **那么** 系统应创建交易记录
- **包括** account_id, stock_code, stock_name, type=BUY, quantity, price, amount, fee
- **并且** 初始状态为 PENDING

#### 场景：创建卖出交易
- **当** 用户提交卖出订单
- **那么** 系统应创建交易记录
- **包括** account_id, stock_code, stock_name, type=SELL, quantity, price, amount, fee
- **并且** 初始状态为 PENDING

### 需求：交易状态流转

系统必须支持交易状态的正确流转。

#### 场景：订单成交
- **当** 挂单交易成交
- **那么** 系统应更新交易状态为 FILLED
- **并且** 记录成交时间

#### 场景：订单撤销
- **当** 用户撤销挂单
- **那么** 系统应更新交易状态为 CANCELLED
- **并且** 记录撤销时间

#### 场景：订单被拒绝
- **当** 订单因余额不足等原因被拒绝
- **那么** 系统应更新交易状态为 REJECTED
- **并且** 在 note 字段记录拒绝原因

### 需求：部分成交处理

系统必须正确处理部分成交的情况。

#### 场景：部分成交发生
- **当** 挂单部分成交（如 100 股成交 50 股）
- **那么** 系统应将原记录状态更新为 OBSOLETE
- **并且** 创建已成交部分的新记录，status=FILLED, parent_id=原记录id
- **并且** 创建剩余部分的新记录，status=PENDING, parent_id=原记录id
- **并且** 调整锁定金额为剩余部分的金额

#### 场景：查询交易历史
- **当** 需要查询交易历史
- **并且** 指定包含部分成交的完整历史
- **那么** 系统应通过 parent_id 关联显示原始记录和拆分后的记录

### 需求：交易查询

系统必须支持多维度查询交易记录。

#### 场景：按账户查询交易
- **当** 按账户 ID 查询交易记录
- **那么** 系统应返回该账户的所有交易
- **并且** 按时间倒序排列

#### 场景：按股票代码查询交易
- **当** 按股票代码查询交易记录
- **那么** 系统应返回该股票的所有交易
- **并且** 包括买卖记录

#### 场景：按交易类型查询
- **当** 按交易类型（BUY/SELL）查询交易记录
- **那么** 系统应返回对应类型的交易记录

### 需求：审计追踪

交易记录本身即作为操作日志用于审计。

#### 场景：完整审计链
- **当** 需要审计某笔交易
- **那么** 系统应提供完整的交易信息
- **包括** 操作时间、操作类型、交易金额、交易状态、关联账户
- **并且** 对于部分成交，可通过 parent_id 追溯原始订单
