# 规格：account-management

账户管理能力，包括用户标识、投入金额、可用金额、锁定金额的管理。

## 增加需求

### 需求：账户创建

系统必须支持创建新账户，记录用户信息和初始资金。

#### 场景：创建新账户
- **当** 用户创建新账户
- **并且** 提供用户标识和投入金额
- **那么** 系统应创建账户记录
- **并且** 初始化 available_amt = 投入金额
- **并且** 初始化 locked_amt = 0
- **并且** 设置状态为 ACTIVE

#### 场景：用户标识唯一性
- **当** 尝试创建已存在的 user_id 账户
- **那么** 系统应拒绝创建
- **并且** 返回错误信息

### 需求：账户查询

系统必须支持按用户标识查询账户信息。

#### 场景：查询现有账户
- **当** 使用有效的 user_id 查询账户
- **那么** 系统应返回完整的账户信息
- **包括** initial_amount, available_amt, locked_amt, status

#### 场景：查询不存在的账户
- **当** 使用不存在的 user_id 查询账户
- **那么** 系统应返回账户不存在错误

### 需求：账户状态管理

系统必须支持账户状态的变更。

#### 场景：冻结账户
- **当** 账户状态从 ACTIVE 变更为 FROZEN
- **那么** 系统应更新账户状态
- **并且** 禁止该账户的任何交易操作

#### 场景：关闭账户
- **当** 账户状态变更为 CLOSED
- **那么** 系统应验证可用金额和锁定金额均为 0
- **并且** 更新账户状态

### 需求：资金计算

系统必须提供准确的资金计算能力。

#### 场景：计算总成本
- **当** 需要计算账户总成本
- **那么** 系统应计算：available_amt + locked_amt + 持仓成本
- **并且** 返回计算结果

#### 场景：计算总市值
- **当** 需要计算账户总市值
- **那么** 系统应计算：available_amt + locked_amt + 持仓市值（实时价格）
- **并且** 返回计算结果
