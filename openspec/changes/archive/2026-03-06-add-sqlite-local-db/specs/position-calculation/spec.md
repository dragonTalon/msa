# 规格：position-calculation

持仓计算，包括持仓数量、持仓成本、持仓市值的计算。

## 增加需求

### 需求：持仓数量计算

系统必须能够计算当前持仓数量。

#### 场景：计算单一股票持仓数量
- **当** 查询某股票的持仓数量
- **那么** 系统应计算：SUM(BUY成交数量) - SUM(SELL成交数量)
- **其中** 仅统计 status=FILLED 的交易
- **并且** 返回净持仓数量（可为负数表示空头）

#### 场景：计算全部持仓
- **当** 查询所有持仓股票
- **那么** 系统应按股票代码分组计算持仓
- **并且** 返回持仓数量大于 0 的股票列表

### 需求：持仓成本计算

系统必须能够计算持仓成本。

#### 场景：计算单一股票持仓成本
- **当** 查询某股票的持仓成本
- **那么** 系统应计算：SUM(BUY成交金额) - SUM(SELL成交金额)
- **其中** 仅统计 status=FILLED 的交易
- **并且** 返回持仓成本金额

#### 场景：计算总持仓成本
- **当** 查询账户的总持仓成本
- **那么** 系统应汇总所有股票的持仓成本
- **并且** 返回总成本金额

### 需求：持仓市值计算

系统必须支持实时查询计算持仓市值。

#### 场景：计算持仓市值
- **当** 需要计算持仓市值
- **那么** 系统应调用 API 获取每只股票的实时价格
- **并且** 计算：SUM(持仓数量 × 实时价格)
- **并且** 返回持仓市值

#### 场景：API 调用失败处理
- **当** 实时价格 API 调用失败
- **那么** 系统应使用最后成功获取的价格
- **或者** 返回错误提示，不进行市值计算

### 需求：盈亏计算

系统必须提供盈亏计算能力。

#### 场景：计算未实现盈亏
- **当** 需要计算未实现盈亏
- **那么** 系统应计算：持仓市值 - 持仓成本
- **并且** 正数表示盈利，负数表示亏损

#### 场景：计算总盈亏
- **当** 需要计算账户总盈亏
- **那么** 系统应计算：总市值 - initial_amount
- **其中** 总市值 = available_amt + locked_amt + 持仓市值

### 需求：持仓查询

系统必须支持查询当前持仓状态。

#### 场景：查询持仓列表
- **当** 查询当前持仓
- **那么** 系统应返回持仓列表
- **包括** 股票代码、股票名称、持仓数量、持仓成本、当前价格、持仓市值、盈亏

#### 场景：查询无持仓
- **当** 账户没有任何持仓
- **那么** 系统应返回空列表
