## 为什么

当前项目通过 API 获取股票数据，但缺乏本地持久化能力。无法记录：
- 用户账户信息和资金状态
- 买卖交易历史
- 持仓情况

添加本地数据库支持后，可以实现：
- 交易记录的持久化存储
- 账户资金的原子性管理
- 持仓成本的准确计算
- 合规性审计追踪

## 变更内容

- 添加 SQLite 数据库依赖
- 创建 `accounts` 表（账户表）
- 创建 `transactions` 表（交易记录表）
- 实现事务原子性保证（账户更新与交易记录在同一事务中）
- 实现金额锁定机制（处理挂单与成交的时间差）

## 能力

### 新增能力

- `sqlite-local-storage`：SQLite 本地数据存储基础能力
- `account-management`：账户管理，包括用户标识、投入金额、可用金额、锁定金额
- `transaction-recording`：交易记录管理，包括买卖记录、状态流转、部分成交处理
- `position-calculation`：持仓计算，包括持仓数量、持仓成本、持仓市值
- `amount-locking`：金额锁定机制，用于挂单时锁定资金

### 修改能力

无

## 影响

**新增依赖**：
- SQLite Go 驱动（如 `github.com/mattn/go-sqlite3` 或 `modernc.org/sqlite`）

**新增代码**：
- `pkg/db/` 目录：数据库初始化、连接管理
- `pkg/model/account.go`：账户数据模型
- `pkg/model/transaction.go`：交易记录数据模型
- 数据库迁移脚本

**受影响模块**：
- 交易相关的逻辑需要调用数据库层
- 可能需要重构现有的 API 调用逻辑以支持数据持久化
