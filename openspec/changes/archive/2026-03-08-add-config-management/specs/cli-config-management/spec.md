# 规格：cli-config-management

CLI 配置管理能力，提供用户友好的命令行配置界面和多种配置方式。

> 📚 **知识上下文**：基于知识库模式 #003 (错误处理和重试策略)
> - 配置加载应该有清晰的错误处理
> - 用户输入验证应该提供友好的错误提示
> - 配置保存失败时应该有明确的反馈

## 增加需求

### 需求：TUI 配置界面

提供交互式 TUI 配置界面，用户可以通过 `msa config` 命令进入配置管理界面。

#### 场景：启动配置界面
- **当** 用户执行 `msa config` 命令
- **那么** 系统应显示 TUI 配置界面
- **并且** 显示当前配置的所有可配置项

#### 场景：显示当前配置
- **当** 用户进入配置界面
- **那么** 系统应显示当前配置值
- **并且** API Key 应隐藏显示（只显示前4后4字符）
- **并且** Provider 应显示友好名称

#### 场景：编辑配置项
- **当** 用户选择编辑某个配置项
- **那么** 系统应进入编辑模式
- **并且** 显示当前值
- **并且** 允许用户输入新值

#### 场景：Provider 选择
- **当** 用户选择 Provider 配置项
- **那么** 系统应显示 Provider 下拉选择器
- **并且** 显示所有已注册的 Provider（友好名称）
- **并且** 标记当前选中的 Provider

#### 场景：切换 Provider
- **当** 用户切换到不同的 Provider
- **那么** 系统应清空 API Key 和 Base URL
- **并且** 填充新 Provider 的默认 Base URL
- **并且** 提示用户重新输入 API Key

#### 场景：保存配置
- **当** 用户选择保存配置
- **并且** 所有配置项验证通过
- **那么** 系统应保存配置到 `~/.msa/msa_config.json`
- **并且** 显示保存成功消息

#### 场景：配置验证失败
- **当** 用户尝试保存配置
- **并且** 存在验证失败的配置项
- **那么** 系统应阻止保存
- **并且** 显示验证失败项和错误信息
- **并且** 高亮显示失败的配置项

#### 场景：重置配置
- **当** 用户选择重置配置
- **那么** 系统应提示确认
- **并且** 确认后将所有配置项重置为默认值

### 需求：命令行参数配置

支持通过命令行参数 `--config` 进行配置。

#### 场景：使用配置文件路径
- **当** 用户执行 `msa --config /path/to/config.json chat`
- **那么** 系统应加载指定的配置文件
- **并且** 该配置优先于默认配置文件

#### 场景：使用 key=value 格式
- **当** 用户执行 `msa --config apikey=sk-xxx chat`
- **那么** 系统应解析 key=value 格式
- **并且** 设置对应的配置项
- **并且** 该配置优先于配置文件和环境变量

#### 场景：多个 --config 参数
- **当** 用户执行 `msa --config apikey=sk-xxx --config loglevel=debug chat`
- **那么** 系统应按顺序处理所有 --config 参数
- **并且** 后面的参数覆盖前面的同名配置

#### 场景：混合使用文件和 key=value
- **当** 用户执行 `msa --config /custom/config.json --config loglevel=debug chat`
- **那么** 系统应先加载配置文件
- **然后** 应用 key=value 覆盖

### 需求：环境变量配置

支持通过环境变量进行配置。

#### 场景：使用环境变量
- **当** 设置 `MSA_API_KEY=sk-xxx` 环境变量
- **并且** 执行 `msa chat`
- **那么** 系统应从环境变量读取 API Key
- **并且** 该配置优先于配置文件

#### 场景：环境变量优先级
- **当** 同时存在命令行参数、环境变量和配置文件
- **那么** 优先级应为：命令行 > 环境变量 > 配置文件 > 默认值

#### 场景：支持的环境变量
- **当** 用户通过环境变量配置
- **那么** 系统应支持以下环境变量：
  - `MSA_PROVIDER` - LLM 提供商
  - `MSA_API_KEY` - API 密钥
  - `MSA_BASE_URL` - API 基础 URL
  - `MSA_LOG_LEVEL` - 日志级别
  - `MSA_LOG_FILE` - 日志文件路径

### 需求：配置持久化

确保配置正确保存和加载。

#### 场景：保存到配置文件
- **当** 用户保存配置
- **那么** 系统应将配置写入 `~/.msa/msa_config.json`
- **并且** 使用 JSON 格式
- **并且** 使用 2 空格缩进

#### 场景：加载配置文件
- **当** 系统启动
- **那么** 系统应尝试加载 `~/.msa/msa_config.json`
- **并且** 如果文件不存在，创建默认配置
- **并且** 如果文件损坏，使用默认配置并记录警告

#### 场景：配置文件权限
- **当** 创建配置文件
- **那么** 文件权限应设置为 `0644`
- **并且** 配置目录权限应设置为 `0755`

### 需求：错误处理

> 📚 **知识上下文**：基于知识库模式 #003 (错误处理和重试策略)

#### 场景：配置保存失败
- **当** 配置保存失败（如权限不足、磁盘满）
- **那么** 系统应显示友好的错误信息
- **并且** 保留当前未保存的更改
- **并且** 允许用户重试或取消

#### 场景：配置文件损坏
- **当** 加载配置文件时发现 JSON 格式错误
- **那么** 系统应记录警告日志
- **并且** 使用默认配置
- **并且** 提示用户配置文件已损坏

#### 场景：无效的命令行参数
- **当** 用户提供的 `--config` 参数格式无效
- **那么** 系统应显示用法说明
- **并且** 退出并返回错误码
