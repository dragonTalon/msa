# 提案：add-knowledge-tools

为股票交易助手添加知识库管理工具，支持 SKILL 系统读写用户画像、策略、错误经验等知识文件。

## 知识上下文

### 相关历史问题

知识库中未找到与"知识库工具"直接相关的问题。

### 需避免的反模式

未识别到特定反模式。

### 推荐模式

✅ **推荐**：参考现有 tools 组件模式：

- **003**：错误处理和重试策略
  - 描述：统一的错误处理方式
  - 使用时机：所有工具实现

- **004**：浏览器管理器单例模式
  - 描述：资源管理采用单例模式
  - 使用时机：文件系统操作可参考

## 风险评估

基于 tools 组件特点，此变更具有以下风险因素：

- **中等**：文件路径注入风险
  - 缓解措施：限制操作目录在 `.msa/knowledge/` 下，禁止路径遍历

- **低**：并发写入冲突
  - 缓解措施：使用文件锁或写入前检查

## 缓解策略

1. 所有文件操作限定在 `.msa/knowledge/` 目录下
2. 文件路径参数进行安全校验，禁止 `..` 等路径遍历
3. 写入操作使用原子写入（先写临时文件，再重命名）

## 为什么

当前 MSA 系统已创建 4 个交易相关 SKILL（morning-analysis、afternoon-trade、market-close-summary、read-error），这些 SKILL 需要：
- 读取用户笔记、策略、画像
- 写入错误经验、交易总结
- 查询 Session 历史记录
- 标记 Session 类型（早盘/午盘/收盘）

但现有工具系统不支持这些操作，导致 SKILL 无法正常执行。

## 变更内容

- 新增 5 个知识库管理工具
- 在 `pkg/logic/tools/` 下创建 knowledge 子包
- 注册工具到 Agent 工具列表

## 能力

### 新增能力

- `read-knowledge-file`：读取知识库文件（笔记、策略、错误等）
- `write-knowledge-file`：写入知识库文件（总结、错误、洞察等）
- `list-knowledge-files`：列出知识库目录下的文件
- `query-sessions-by-date`：按日期查询 Session 记录
- `add-session-tag`：为 Session 添加标签

### 修改能力

无（均为新增能力）

## 影响

- 新增代码：`pkg/logic/tools/knowledge/` 目录
- 修改文件：`pkg/logic/agent/chat.go`（注册新工具）
- 依赖：无新增外部依赖
- 兼容性：完全向后兼容，不影响现有功能

## 非目标

- 不实现知识文件的版本控制
- 不实现知识文件的全文搜索
- 不实现跨 Session 的复杂关联查询
- 不修改现有 SKILL 的执行逻辑
