# 规格：gorm-orm

> 📚 **知识上下文**：基于 `add-sqlite-local-db` 变更实现中的问题
>
> **历史问题**：
> - **测试超时**: 原生 SQL 配置导致服务层测试超时
> - **并发问题**: SQLite 连接配置不当导致事务处理超时
> - **代码冗长**: 需要手写大量 SQL 语句
> - **维护成本高**: 表结构变更需要修改多处 SQL 代码

## 增加需求

### [R-1] GORM 框架集成

系统必须集成 GORM 框架以替代原生 SQL 操作。

#### 场景：添加 GORM 依赖
- **当** 项目添加 GORM 依赖
- **那么** 必须包含 `gorm.io/gorm` 和 `gorm.io/driver/sqlite`

#### 场景：验证依赖可用性
- **当** 导入 GORM 包
- **那么** 必须能够成功编译并初始化 GORM 实例

---

### [R-2] GORM 数据库初始化

系统必须使用 GORM 的 API 进行数据库初始化。

#### 场景：初始化数据库连接
- **当** 调用初始化函数
- **那么** 使用 `gorm.Open()` 建立连接
- **并且** 返回 `*gorm.DB` 实例而非 `*sql.DB`

#### 场景：配置 SQLite 连接
- **当** 打开数据库连接
- **那么** 必须使用 GORM 的 SQLite driver
- **并且** 配置正确的连接池参数以避免测试超时

#### 场景：数据库目录不存在
- **当** `~/.msa` 目录不存在
- **那么** 自动创建该目录
- **并且** 返回有效的数据库连接

---

### [R-3] 自动迁移

系统必须使用 GORM 的 AutoMigrate 功能进行表结构管理。

#### 场景：执行数据库迁移
- **当** 调用迁移函数
- **那么** 使用 `db.AutoMigrate()` 自动创建/更新表结构
- **并且** 基于模型结构体定义自动生成 SQL

#### 场景：添加新字段
- **当** 模型结构体添加新字段
- **那么** AutoMigrate 自动添加对应的数据库列
- **并且** 不需要手写 ALTER TABLE 语句

---

### [R-4] 模型定义

所有数据模型必须包含 GORM 标签以支持 ORM 映射。

#### 场景：Account 模型定义
- **当** 定义 Account 结构体
- **那么** 必须嵌入 `gorm.Model`
- **并且** 使用 `gorm:"type:TEXT"` 等标签指定字段类型
- **并且** 使用 `gorm:"uniqueIndex"` 标签定义唯一索引

#### 场景：Transaction 模型定义
- **当** 定义 Transaction 结构体
- **那么** 必须包含外键关系标签
- **并且** 使用 `gorm:"foreignKey:AccountID"` 指定外键
- **并且** 使用 `gorm:"constraint:OnDelete:CASCADE"` 配置级联删除

---

### [R-5] CRUD 操作封装

系统必须提供基于 GORM 的 CRUD 操作函数。

#### 场景：创建记录
- **当** 调用 CreateAccount
- **那么** 使用 `db.Create(&account)` 插入记录
- **并且** 自动处理 ID 和时间戳字段

#### 场景：查询单条记录
- **当** 根据 ID 查询记录
- **那么** 使用 `db.First(&account, id)` 或 `db.Where("user_id = ?", userID).First(&account)`
- **并且** 使用 GORM 的错误处理机制

#### 场景：更新记录
- **当** 更新记录字段
- **那么** 使用 `db.Model(&account).Update("status", status)` 或 `db.Updates(&account)`
- **并且** 自动更新 `updated_at` 字段

#### 场景：删除记录
- **当** 删除记录（如需要）
- **那么** 使用 `db.Delete(&account)` 软删除（如果使用 gorm.Model）
- **或者** 使用 `db.Unscoped().Delete(&account)` 硬删除

---

### [R-6] 事务处理

系统必须使用 GORM 的事务 API 进行事务管理。

#### 场景：开始事务
- **当** 需要执行多个原子操作
- **那么** 使用 `db.Begin()` 创建事务
- **并且** 返回 `*gorm.DB` 事务实例

#### 场景：提交事务
- **当** 所有操作成功完成
- **那么** 调用 `tx.Commit()` 提交事务

#### 场景：回滚事务
- **当** 操作过程中发生错误
- **那么** 调用 `tx.Rollback()` 回滚事务
- **并且** 使用 `defer tx.Rollback()` 确保异常时回滚

#### 场景：事务中的数据库操作
- **当** 在事务中执行数据库操作
- **那么** 使用事务实例 `tx` 而非原始的 `db`
- **并且** 所有操作使用同一个事务实例

---

### [R-7] 查询构建器

系统必须使用 GORM 的链式查询构建器进行复杂查询。

#### 场景：条件查询
- **当** 根据多个条件查询交易记录
- **那么** 使用 `db.Where("account_id = ?", accountID).Where("status = ?", status).Find(&transactions)`

#### 场景：聚合查询
- **当** 计算持仓数量
- **那么** 使用 `db.Model(&Transaction{}).Select("COALESCE(SUM(quantity), 0)").Where(...).Scan(&position)`

#### 场景：分组查询
- **当** 按股票代码分组统计
- **那么** 使用 `db.Model(&Transaction{}).Select("stock_code, SUM(quantity)").Group("stock_code").Find(&results)`

---

### [R-8] 钩子函数

系统必须使用 GORM 钩子函数处理模型生命周期事件。

#### 场景：创建前验证
- **当** 创建 Account 记录前
- **那么** 使用 `BeforeCreate` 钩子验证 user_id 唯一性
- **或者** 使用 `BeforeSave` 钩子验证金额字段

#### 场景：更新后处理
- **当** 更新 Transaction 状态后
- **那么** 使用 `AfterUpdate` 钩子触发相关业务逻辑（如通知）

---

### [R-9] 错误处理

系统必须正确处理 GORM 返回的错误。

#### 场景：记录不存在
- **当** 查询不存在的记录
- **那么** 返回 `gorm.ErrRecordNotFound` 错误
- **并且** 调用方处理该错误情况

#### 场景：唯一约束冲突
- **当** 插入重复的 user_id
- **那么** 返回唯一约束错误
- **并且** 错误信息包含相关字段信息

#### 场景：事务错误
- **当** 事务中任何操作失败
- **那么** 返回错误并触发回滚
- **并且** 不提交部分事务

---

### [R-10] 测试支持

系统必须提供支持测试的数据库初始化方法。

#### 场景：使用内存数据库测试
- **当** 运行单元测试
- **那么** 支持使用 `:memory:` 作为数据库路径
- **并且** 每个测试使用独立的数据库实例

#### 场景：使用临时文件测试
- **当** 运行集成测试
- **那么** 支持指定自定义数据库路径
- **并且** 测试结束后清理临时文件

---

> 📚 **历史问题解决**：
>
> 通过使用 GORM，预期解决以下问题：
> 1. **测试超时**: GORM 自动管理连接池，配置更合理
> 2. **并发问题**: GORM 的连接池机制避免连接竞争
> 3. **代码冗长**: 使用 ORM 减少约 40-50% 的代码量
> 4. **维护成本**: AutoMigrate 自动处理表结构变更
