# 规格：account-management

> 📚 **知识上下文**：基于提案中的错误处理和重试策略推荐模式
>
> **推荐模式**：外部 API 调用失败时应有适当的错误处理和降级策略

## 增加需求

### [R-1] 创建账户

**场景：创建第一个账户**
- **当** 数据库中不存在 ACTIVE 状态的账户
- **且** 用户提供的初始金额有效（≥ 0）
- **那么** 创建新账户，user_id 为 "default"，状态为 ACTIVE
- **且** 返回账户信息（ID、初始金额、可用余额、状态）

**场景：重复创建账户被拒绝**
- **当** 数据库中已存在 ACTIVE 状态的账户
- **那么** 返回错误，提示已存在活跃账户
- **且** 不创建新账户

**场景：数据库不可用时降级**
- **当** 数据库未初始化或连接失败
- **那么** 返回友好的错误提示
- **且** 不导致程序崩溃

### [R-2] 查询账户

**场景：查询活跃账户**
- **当** 用户调用查询账户 tool
- **且** 数据库中存在 ACTIVE 状态的账户
- **那么** 返回账户完整信息
  - 账户 ID
  - 初始金额（元）
  - 可用余额（元）
  - 锁定金额（元）
  - 账户状态
  - 创建时间

**场景：无活跃账户时提示**
- **当** 数据库中不存在 ACTIVE 状态的账户
- **那么** 返回友好的提示信息
- **且** 建议用户先创建账户

### [R-3] 修改账户状态

**场景：冻结账户**
- **当** 用户执行冻结操作
- **且** 账户当前状态为 ACTIVE
- **那么** 将账户状态更新为 FROZEN
- **且** 返回操作成功提示

**场景：解冻账户**
- **当** 用户执行解冻操作
- **且** 账户当前状态为 FROZEN
- **那么** 将账户状态更新为 ACTIVE
- **且** 返回操作成功提示

**场景：关闭账户**
- **当** 用户执行关闭操作
- **且** 账户当前状态为 ACTIVE 或 FROZEN
- **且** 可用余额为 0 且锁定金额为 0
- **那么** 将账户状态更新为 CLOSED
- **且** 返回操作成功提示

**场景：关闭余额非零账户被拒绝**
- **当** 用户执行关闭操作
- **且** 账户可用余额或锁定金额不为 0
- **那么** 返回错误提示
- **且** 不修改账户状态

### [R-4] 金额格式化

**场景：显示金额**
- **当** 需要向用户展示金额信息
- **那么** 将分转换为元（÷ 100）
- **且** 保留两位小数
- **且** 格式化为易读的字符串（如 "12345.00"）
