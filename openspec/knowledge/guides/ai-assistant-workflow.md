# AI 助手工作流程指南

## 指南标识

- **ID**: ai-assistant-workflow
- **类别**: 工作流程
- **来源**: add-web-search-tool 实施总结

## 适用场景

本指南适用于 AI 助手辅助代码开发的场景，定义了正确的工作流程。

## 完整工作流程

### 阶段 1：需求理解

1. **理解用户需求**
   - 询问澄清问题（如果有歧义）
   - 确认功能范围
   - 识别技术约束

2. **制定实现方案**
   - 提出技术方案
   - 评估风险和复杂度
   - 与用户确认方案

### 阶段 2：代码实现

1. **编写代码**
   - 按照最佳实践编写
   - 添加必要的注释
   - 保持代码简洁

2. **编译验证**
   - 确保代码编译通过
   - 检查编译警告
   - 运行基本测试

3. **⏸️ 暂停等待审查**

   **关键步骤！** 代码完成后，**暂停**并等待用户审查。

   ```bash
   ✅ 代码已实现
   ✅ 编译通过
   ⏸️  等待用户审查...
   ```

### 阶段 3：用户审查

1. **用户检查代码**
   - 代码质量
   - 功能正确性
   - 安全性
   - 性能考虑

2. **用户测试**
   - 运行测试
   - 验证功能
   - 检查边界情况

3. **反馈和修正**
   - 用户提出修改意见
   - AI 助手进行修正
   - 重新编译验证

### 阶段 4：提交（由用户执行）

1. **用户确认后手动提交**
   ```bash
   git add <files>
   git commit -m "<message>"
   ```

2. **AI 助手可以提供提交命令**
   ```bash
   # 建议的提交命令：
   git add pkg/logic/tools/search/
   git commit -m "feat: 添加网页搜索工具"
   ```

   **但不要执行！**

## 正确做法示例

### ✅ 正确

```
AI: 代码实现完成，编译通过。
    请审查以下文件：
    - pkg/logic/tools/search/search.go
    - pkg/logic/tools/search/google.go

    建议的提交命令：
    git add pkg/logic/tools/search/
    git commit -m "feat: 添加网页搜索和内容抓取工具"

    等待您的审查和确认。
```

### ❌ 错误

```
AI: 代码实现完成，正在创建 Git 提交...
    [自动执行 git commit]
    提交已创建：abc123
```

## 行为准则

### AI 助手应该做

- ✅ 实现代码
- ✅ 编译验证
- ✅ 提供测试建议
- ✅ 提供提交命令
- ✅ 等待用户确认
- ✅ 根据反馈修改

### AI 助手不应该做

- ❌ 自动执行 `git commit`
- ❌ 自动执行 `git push`
- ❌ 跳过审查阶段
- ❌ 假设代码完美无需审查

## 模板

### 完成代码时

```
✅ 代码实现完成

文件列表：
- <file1>
- <file2>

编译状态：通过

下一步：请审查代码并确认是否可以提交。
```

### 用户提供反馈后

```
收到反馈，正在修改...

修改内容：
- <change1>
- <change2>

重新编译：通过

请再次审查。
```

### 用户确认后

```
感谢确认！

您可以使用以下命令提交：
git add <files>
git commit -m "<message>"

需要我帮您执行其他操作吗？
```
