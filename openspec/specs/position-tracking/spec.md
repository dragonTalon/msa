# 规格：position-tracking

持仓跟踪能力，提供持仓查询、账户总览和实时价格计算功能。

## Purpose

为 MSA 系统提供持仓跟踪能力，用户可以查询当前持仓列表、账户总览，系统自动计算盈亏。当外部价格 API 失败时有降级策略，确保基本功能可用。

## Requirements

### Requirement: 获取所有持仓

系统 SHALL 提供持仓查询功能，返回所有持仓股票信息。

#### Scenario: 查询持仓列表

- **WHEN** 用户调用持仓查询 tool
- **AND** 数据库中存在活跃账户
- **AND** 账户有持仓记录
- **THEN** 返回所有持仓股票信息
  - 股票代码
  - 股票名称
  - 持仓数量
  - 持仓成本（元）
  - 当前价格（元）
  - 持仓市值（元）
  - 盈亏金额（元）
  - 盈亏比例（%）

#### Scenario: 无持仓时返回空列表

- **WHEN** 用户调用持仓查询 tool
- **AND** 账户没有任何持仓
- **THEN** 返回空列表
- **AND** 提示用户当前无持仓

#### Scenario: 价格获取失败时降级

- **WHEN** 调用股票价格 API 失败
- **THEN** 仍然返回持仓列表
- **AND** 显示持仓数量和成本
- **AND** 当前价格、市值、盈亏显示为 "N/A" 或 "-"
- **AND** 标注价格数据获取失败

### Requirement: 获取账户总览

系统 SHALL 提供账户总览查询功能，返回账户摘要信息。

#### Scenario: 查询账户摘要

- **WHEN** 用户调用账户总览 tool
- **AND** 数据库中存在活跃账户
- **THEN** 返回账户摘要信息
  - 总资产（元）= 可用余额 + 持仓市值
  - 可用余额（元）
  - 锁定金额（元）
  - 持仓市值（元）
  - 总成本（元）= 初始金额
  - 总盈亏（元）= 总资产 - 初始金额
  - 盈亏比例（%）= 总盈亏 / 初始金额 × 100

#### Scenario: 盈利显示

- **WHEN** 总盈亏为正数
- **THEN** 盈亏金额显示绿色或加 "+" 前缀
- **AND** 盈亏比例显示绿色

#### Scenario: 亏损显示

- **WHEN** 总盈亏为负数
- **THEN** 盈亏金额显示红色
- **AND** 盈亏比例显示红色

### Requirement: 实时价格获取

系统 SHALL 复用现有接口批量获取股票实时价格用于持仓计算。

#### Scenario: 批量获取股票价格

- **WHEN** 需要计算持仓市值
- **AND** 持仓包含多只股票
- **THEN** 复用 `stock/fetchStockData()` 获取每只股票的实时价格
- **AND** 将价格字符串转换为分（int64）

#### Scenario: 单只股票价格获取失败

- **WHEN** 某只股票的价格 API 调用失败
- **THEN** 跳过该股票的价格计算
- **AND** 继续处理其他股票
- **AND** 在结果中标注该股票价格不可用

#### Scenario: 股票代码格式转换

- **WHEN** 从交易记录获取的股票代码格式与 API 要求不一致
- **THEN** 自动转换为 API 支持的格式
- **AND** 记录转换日志便于调试

### Requirement: 持仓计算逻辑

系统 SHALL 根据已成交交易记录正确计算持仓数量和成本。

#### Scenario: 计算持仓数量

- **WHEN** 计算某股票的持仓数量
- **THEN** 持仓数量 = 已成交买入数量 - 已成交卖出数量
- **AND** 只计算状态为 FILLED 的交易

#### Scenario: 计算持仓成本

- **WHEN** 计算某股票的持仓成本
- **THEN** 持仓成本 = 已成交买入金额（含手续费） - 已成交卖出净收入
- **AND** 买入金额 = SUM(quantity × price + fee)
- **AND** 卖出净收入 = SUM(quantity × price - fee)

#### Scenario: 零持仓不显示

- **WHEN** 某股票的持仓数量 ≤ 0
- **THEN** 不在持仓列表中显示该股票
- **AND** 该股票可能已全部卖出
