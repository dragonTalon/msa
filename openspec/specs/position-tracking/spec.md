# 规格：position-tracking

持仓跟踪能力，提供持仓查询、账户总览和实时价格计算功能。

## Purpose

为 MSA 系统提供持仓跟踪能力，用户可以查询当前持仓列表、账户总览，系统自动计算盈亏。当外部价格 API 失败时有降级策略，确保基本功能可用。

## Requirements

### [R-1] 获取所有持仓

**场景：查询持仓列表**
- **当** 用户调用持仓查询 tool
- **且** 数据库中存在活跃账户
- **且** 账户有持仓记录
- **那么** 返回所有持仓股票信息
  - 股票代码
  - 股票名称
  - 持仓数量
  - 持仓成本（元）
  - 当前价格（元）
  - 持仓市值（元）
  - 盈亏金额（元）
  - 盈亏比例（%）

**场景：无持仓时返回空列表**
- **当** 用户调用持仓查询 tool
- **且** 账户没有任何持仓
- **那么** 返回空列表
- **且** 提示用户当前无持仓

**场景：价格获取失败时降级**
- **当** 调用股票价格 API 失败
- **那么** 仍然返回持仓列表
- **且** 显示持仓数量和成本
- **且** 当前价格、市值、盈亏显示为 "N/A" 或 "-"
- **且** 标注价格数据获取失败

### [R-2] 获取账户总览

**场景：查询账户摘要**
- **当** 用户调用账户总览 tool
- **且** 数据库中存在活跃账户
- **那么** 返回账户摘要信息
  - 总资产（元）= 可用余额 + 持仓市值
  - 可用余额（元）
  - 锁定金额（元）
  - 持仓市值（元）
  - 总成本（元）= 初始金额
  - 总盈亏（元）= 总资产 - 初始金额
  - 盈亏比例（%）= 总盈亏 / 初始金额 × 100

**场景：盈利显示**
- **当** 总盈亏为正数
- **那么** 盈亏金额显示绿色或加 "+" 前缀
- **且** 盈亏比例显示绿色

**场景：亏损显示**
- **当** 总盈亏为负数
- **那么** 盈亏金额显示红色
- **且** 盈亏比例显示红色

### [R-3] 实时价格获取

**场景：批量获取股票价格**
- **当** 需要计算持仓市值
- **且** 持仓包含多只股票
- **那么** 复用 `stock/fetchStockData()` 获取每只股票的实时价格
- **且** 将价格字符串转换为分（int64）

**场景：单只股票价格获取失败**
- **当** 某只股票的价格 API 调用失败
- **那么** 跳过该股票的价格计算
- **且** 继续处理其他股票
- **且** 在结果中标注该股票价格不可用

**场景：股票代码格式转换**
- **当** 从交易记录获取的股票代码格式与 API 要求不一致
- **那么** 自动转换为 API 支持的格式
- **且** 记录转换日志便于调试

### [R-4] 持仓计算逻辑

**场景：计算持仓数量**
- **当** 计算某股票的持仓数量
- **那么** 持仓数量 = 已成交买入数量 - 已成交卖出数量
- **且** 只计算状态为 FILLED 的交易

**场景：计算持仓成本**
- **当** 计算某股票的持仓成本
- **那么** 持仓成本 = 已成交买入金额（含手续费） - 已成交卖出净收入
- **且** 买入金额 = SUM(quantity × price + fee)
- **且** 卖出净收入 = SUM(quantity × price - fee)

**场景：零持仓不显示**
- **当** 某股票的持仓数量 ≤ 0
- **那么** 不在持仓列表中显示该股票
- **且** 该股票可能已全部卖出
