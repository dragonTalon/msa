# 规格：position-calculation

## Purpose

系统 SHALL 提供持仓计算能力，包括持仓数量、成本、市值和盈亏计算。

## Requirements

### Requirement: 持仓数量计算

系统 SHALL 能够计算用户在特定股票上的持仓数量。

#### Scenario: 计算持仓数量
- **WHEN** 查询用户在某只股票上的持仓
- **THEN** 系统计算：买入成交数量 - 卖出成交数量
- **AND** 只统计已成交（FILLED 状态）的交易
- **AND** 返回持仓数量（可能为负或零）

#### Scenario: 无持仓股票
- **WHEN** 查询从未交易过的股票
- **THEN** 系统返回持仓数量为 0
- **AND** 不返回错误

---

### Requirement: 持仓成本计算

系统 SHALL 能够计算持仓的成本基础。

#### Scenario: 计算持仓成本
- **WHEN** 查询某只股票的持仓成本
- **THEN** 系统计算：买入总支出（含手续费） - 卖出总收入（净收入）
- **AND** 只统计已成交的交易
- **AND** 返回持仓成本（分为单位）

#### Scenario: 零持仓
- **WHEN** 股票已全部卖出（买入 = 卖出）
- **THEN** 系统返回持仓成本为 0
- **AND** 返回已实现盈亏

---

### Requirement: 持仓市值计算

系统 SHALL 能够计算持仓的当前市值。

#### Scenario: 计算持仓市值
- **WHEN** 提供股票的当前价格
- **THEN** 系统计算：持仓数量 × 当前价格
- **AND** 返回市值（分为单位）
- **AND** 当前价格由外部价格 API 提供

#### Scenario: 无持仓
- **WHEN** 持仓数量为 0
- **THEN** 系统返回市值为 0

---

### Requirement: 账户总资产计算

系统 SHALL 能够计算账户的总资产和盈亏情况。

#### Scenario: 计算账户总成本
- **WHEN** 查询账户的总投入成本
- **THEN** 系统计算：初始金额 + 所有买入支出
- **AND** 等价计算：可用金额 + 锁定金额 + 持仓成本

#### Scenario: 计算账户总市值
- **WHEN** 提供所有持仓股票的当前价格
- **THEN** 系统计算：可用金额 + 锁定金额 + 所有持仓市值
- **AND** 返回总市值（分为单位）

#### Scenario: 计算账户盈亏
- **WHEN** 查询账户的盈亏情况
- **THEN** 系统计算：当前总市值 - 初始投入金额
- **AND** 返回盈亏金额（分为单位）

---

### Requirement: 获取所有持仓

系统 SHALL 能够获取用户的所有持仓汇总信息。

#### Scenario: 获取持仓列表
- **WHEN** 查询用户的所有持仓
- **THEN** 系统返回每只股票的持仓信息
- **AND** 包含：股票代码、股票名称、持仓数量、持仓成本、当前价格、市值、盈亏
- **AND** 过滤掉持仓数量为 0 的股票

#### Scenario: 实时价格依赖
- **WHEN** 计算市值和盈亏时
- **THEN** 系统需要从外部获取实时价格
- **AND** 使用用户提供的最新价格
- **AND** 如果某股票没有价格信息，市值显示为 0 或空值

---

### Requirement: 计算精度

系统 MUST 确保计算的准确性和精度。

#### Scenario: 金额精度
- **WHEN** 计算金额相关的数值时
- **THEN** 系统使用分为单位（整数）进行计算
- **AND** 避免浮点数精度问题

#### Scenario: 大额计算
- **WHEN** 计算涉及大量交易记录时
- **THEN** 系统必须能够高效聚合计算
- **AND** 在合理时间内返回结果
