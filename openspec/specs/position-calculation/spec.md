# 规格：position-calculation

> **能力**: 持仓计算
> **目的**: 系统必须提供持仓计算能力，包括持仓数量、成本、市值和盈亏计算。

## 需求：持仓数量计算

系统必须能够计算用户在特定股票上的持仓数量。

### 场景：计算持仓数量
- **当** 查询用户在某只股票上的持仓
- **那么** 系统计算：买入成交数量 - 卖出成交数量
- **并且** 只统计已成交（FILLED 状态）的交易
- **并且** 返回持仓数量（可能为负或零）

### 场景：无持仓股票
- **当** 查询从未交易过的股票
- **那么** 系统返回持仓数量为 0
- **并且** 不返回错误

---

## 需求：持仓成本计算

系统必须能够计算持仓的成本基础。

### 场景：计算持仓成本
- **当** 查询某只股票的持仓成本
- **那么** 系统计算：买入总支出（含手续费） - 卖出总收入（净收入）
- **并且** 只统计已成交的交易
- **并且** 返回持仓成本（分为单位）

### 场景：零持仓
- **当** 股票已全部卖出（买入 = 卖出）
- **那么** 系统返回持仓成本为 0
- **或者** 返回已实现盈亏

---

## 需求：持仓市值计算

系统必须能够计算持仓的当前市值。

### 场景：计算持仓市值
- **当** 提供股票的当前价格
- **那么** 系统计算：持仓数量 × 当前价格
- **并且** 返回市值（分为单位）
- **并且** 当前价格由外部价格 API 提供

### 场景：无持仓
- **当** 持仓数量为 0
- **那么** 系统返回市值为 0

---

## 需求：账户总资产计算

系统必须能够计算账户的总资产和盈亏情况。

### 场景：计算账户总成本
- **当** 查询账户的总投入成本
- **那么** 系统计算：初始金额 + 所有买入支出
- **或者** 等价计算：可用金额 + 锁定金额 + 持仓成本

### 场景：计算账户总市值
- **当** 提供所有持仓股票的当前价格
- **那么** 系统计算：可用金额 + 锁定金额 + 所有持仓市值
- **并且** 返回总市值（分为单位）

### 场景：计算账户盈亏
- **当** 查询账户的盈亏情况
- **那么** 系统计算：当前总市值 - 初始投入金额
- **并且** 返回盈亏金额（分为单位）

---

## 需求：获取所有持仓

系统必须能够获取用户的所有持仓汇总信息。

### 场景：获取持仓列表
- **当** 查询用户的所有持仓
- **那么** 系统返回每只股票的持仓信息
- **并且** 包含：股票代码、股票名称、持仓数量、持仓成本、当前价格、市值、盈亏
- **并且** 过滤掉持仓数量为 0 的股票

### 场景：实时价格依赖
- **当** 计算市值和盈亏时
- **那么** 系统需要从外部获取实时价格
- **或者** 使用用户提供的最新价格
- **并且** 如果某股票没有价格信息，市值显示为 0 或空值

---

## 需求：计算精度

系统必须确保计算的准确性和精度。

### 场景：金额精度
- **当** 计算金额相关的数值时
- **那么** 系统使用分为单位（整数）进行计算
- **并且** 避免浮点数精度问题

### 场景：大额计算
- **当** 计算涉及大量交易记录时
- **那么** 系统必须能够高效聚合计算
- **并且** 在合理时间内返回结果
