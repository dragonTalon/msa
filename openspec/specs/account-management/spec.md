# 规格：account-management

> **能力**: 账户管理
> **目的**: 系统必须提供用户账户的管理能力，包括账户创建、查询、状态管理和资金管理。

## 需求：创建账户

系统必须支持为用户创建交易账户。

### 场景：创建新账户
- **当** 新用户注册或首次使用系统
- **并且** 提供用户 ID 和初始投入金额
- **那么** 系统必须创建新账户
- **并且** 初始化可用金额为投入金额，锁定金额为 0
- **并且** 账户状态为 ACTIVE
- **并且** 返回账户 ID

### 场景：用户 ID 唯一性
- **当** 尝试创建已存在的用户 ID 的账户
- **那么** 系统必须返回唯一约束冲突错误
- **并且** 错误信息包含用户 ID

### 场景：金额单位
- **当** 创建账户时
- **那么** 金额必须以分为单位存储（整数）
- **并且** 显示时转换为元（保留 2 位小数）

---

## 需求：查询账户

系统必须支持按用户 ID 或账户 ID 查询账户信息。

### 场景：按用户 ID 查询
- **当** 用户提供用户 ID
- **那么** 系统返回对应的账户信息
- **并且** 包含：ID、用户 ID、初始金额、可用金额、锁定金额、状态、创建时间

### 场景：按账户 ID 查询
- **当** 系统内部使用账户 ID 查询
- **那么** 系统返回对应的账户信息

### 场景：账户不存在
- **当** 查询不存在的账户
- **那么** 系统返回明确的错误信息

---

## 需求：更新账户状态

系统必须支持更新账户状态（正常、冻结、关闭）。

### 场景：冻结账户
- **当** 管理员或系统要求冻结账户
- **那么** 系统将账户状态更新为 FROZEN
- **并且** 冻结期间禁止交易操作

### 场景：解冻账户
- **当** 管理员解除账户冻结
- **那么** 系统将账户状态更新为 ACTIVE
- **并且** 恢复交易操作权限

### 场景：关闭账户
- **当** 用户申请关闭账户
- **那么** 系统验证可用金额和锁定金额均为 0
- **并且** 如果验证通过，将账户状态更新为 CLOSED
- **并且** 如果验证失败，返回错误说明原因

---

## 需求：账户金额管理

系统必须支持在事务中原子性地更新账户金额。

### 场景：锁定金额（买入订单）
- **当** 用户提交买入订单
- **那么** 系统减少可用金额，增加锁定金额
- **并且** 操作必须在事务中执行
- **并且** 更新是原子的（要么全部成功，要么全部失败）

### 场景：解锁金额（买入成交）
- **当** 买入订单成交
- **那么** 系统减少锁定金额（已成交部分）
- **并且** 可用金额保持不变（已在提交时扣除）

### 场景：返还锁定（买入撤销）
- **当** 买入订单被撤销
- **那么** 系统减少锁定金额，增加等额可用金额
- **并且** 操作必须在事务中执行

### 场景：增加可用金额（卖出成交）
- **当** 卖出订单成交
- **那么** 系统增加可用金额（成交金额 - 手续费）
- **并且** 卖出订单不锁定金额

---

## 需求：账户列表和查询

系统必须支持查询多个账户。

### 场景：获取所有账户
- **当** 管理员请求账户列表
- **那么** 系统返回所有账户
- **并且** 按创建时间倒序排列

### 场景：按状态筛选账户
- **当** 管理员请求特定状态的账户列表
- **那么** 系统只返回该状态的账户
