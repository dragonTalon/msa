# 规格：account-management

## Purpose

系统 SHALL 提供用户账户的管理能力，包括账户创建、查询、状态管理和资金管理。

## Requirements

### Requirement: 创建账户

系统 SHALL 支持为用户创建交易账户。

#### Scenario: 创建新账户
- **WHEN** 新用户注册或首次使用系统
- **AND** 提供用户 ID 和初始投入金额
- **THEN** 系统必须创建新账户
- **AND** 初始化可用金额为投入金额，锁定金额为 0
- **AND** 账户状态为 ACTIVE
- **AND** 返回账户 ID

#### Scenario: 用户 ID 唯一性
- **WHEN** 尝试创建已存在的用户 ID 的账户
- **THEN** 系统必须返回唯一约束冲突错误
- **AND** 错误信息包含用户 ID

#### Scenario: 金额单位
- **WHEN** 创建账户时
- **THEN** 金额必须以毫为单位存储（整数）
- **AND** 显示时转换为元（保留 2 位小数）

---

### Requirement: 查询账户

系统 SHALL 支持按用户 ID 或账户 ID 查询账户信息。

#### Scenario: 按用户 ID 查询
- **WHEN** 用户提供用户 ID
- **THEN** 系统返回对应的账户信息
- **AND** 包含：ID、用户 ID、初始金额、可用金额、锁定金额、状态、创建时间

#### Scenario: 按账户 ID 查询
- **WHEN** 系统内部使用账户 ID 查询
- **THEN** 系统返回对应的账户信息

#### Scenario: 账户不存在
- **WHEN** 查询不存在的账户
- **THEN** 系统返回明确的错误信息

---

### Requirement: 更新账户状态

系统 SHALL 支持更新账户状态（正常、冻结、关闭）。

#### Scenario: 冻结账户
- **WHEN** 管理员或系统要求冻结账户
- **THEN** 系统将账户状态更新为 FROZEN
- **AND** 冻结期间禁止交易操作

#### Scenario: 解冻账户
- **WHEN** 管理员解除账户冻结
- **THEN** 系统将账户状态更新为 ACTIVE
- **AND** 恢复交易操作权限

#### Scenario: 关闭账户
- **WHEN** 用户申请关闭账户
- **THEN** 系统验证可用金额和锁定金额均为 0
- **AND** 如果验证通过，将账户状态更新为 CLOSED
- **AND** 如果验证失败，返回错误说明原因

---

### Requirement: 账户金额管理

系统 SHALL 支持在事务中原子性地更新账户金额。

#### Scenario: 锁定金额（买入订单）
- **WHEN** 用户提交买入订单
- **THEN** 系统减少可用金额，增加锁定金额
- **AND** 操作必须在事务中执行
- **AND** 更新是原子的（要么全部成功，要么全部失败）

#### Scenario: 解锁金额（买入成交）
- **WHEN** 买入订单成交
- **THEN** 系统减少锁定金额（已成交部分）
- **AND** 可用金额保持不变（已在提交时扣除）

#### Scenario: 返还锁定（买入撤销）
- **WHEN** 买入订单被撤销
- **THEN** 系统减少锁定金额，增加等额可用金额
- **AND** 操作必须在事务中执行

#### Scenario: 增加可用金额（卖出成交）
- **WHEN** 卖出订单成交
- **THEN** 系统增加可用金额（成交金额 - 手续费）
- **AND** 卖出订单不锁定金额

---

### Requirement: 账户列表和查询

系统 SHALL 支持查询多个账户。

#### Scenario: 获取所有账户
- **WHEN** 管理员请求账户列表
- **THEN** 系统返回所有账户
- **AND** 按创建时间倒序排列

#### Scenario: 按状态筛选账户
- **WHEN** 管理员请求特定状态的账户列表
- **THEN** 系统只返回该状态的账户
