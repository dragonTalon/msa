# 规格：read-knowledge-file

> **能力**: 读取知识文件
> **目的**: 系统必须提供工具来读取知识库目录下的文件，用于 SKILL 获取用户笔记、策略、错误经验等信息。

## 需求：安全读取知识文件

系统必须提供工具来读取知识库目录下的文件，用于 SKILL 获取用户笔记、策略、错误经验等信息。

### 场景：读取存在的知识文件
- **当** 调用工具读取 `.msa/knowledge/notes/2026-03-14.md`
- **那么** 返回文件内容和元数据（YAML frontmatter 解析后的结构）

### 场景：读取不存在的文件
- **当** 调用工具读取不存在的文件路径
- **那么** 返回错误信息，明确标识文件不存在

### 场景：读取非知识库目录的文件
- **当** 调用工具读取 `/etc/passwd` 或 `../../../etc/passwd`
- **那么** 返回错误信息，拒绝访问（路径遍历攻击防护）

---

## 需求：路径安全校验

所有文件路径必须经过安全校验，确保只能访问 `.msa/knowledge/` 目录下的文件。

### 场景：拒绝路径遍历
- **当** 请求路径包含 `..` 或绝对路径前缀
- **那么** 返回错误 "path traversal not allowed"

### 场景：限制访问范围
- **当** 请求路径为 `notes/2026-03-14.md`
- **那么** 自动转换为 `.msa/knowledge/notes/2026-03-14.md`

---

## 需求：YAML Frontmatter 解析

知识文件通常包含 YAML frontmatter，系统必须解析并返回结构化数据。

### 场景：解析带 frontmatter 的文件
- **当** 文件内容以 `---` 开头
- **那么** 解析 YAML 元数据和 Markdown 正文，分别返回

### 场景：处理无 frontmatter 的文件
- **当** 文件不包含 YAML frontmatter
- **那么** 返回空元数据和完整正文内容
