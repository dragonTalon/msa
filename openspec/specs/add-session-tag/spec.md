# 规格：add-session-tag

> **能力**: 添加会话标签
> **目的**: 系统必须提供工具来为 Session 添加标签，用于区分早盘、午盘、收盘等不同类型的会话。

## 需求：为 Session 添加标签

系统必须提供工具来为 Session 添加标签，用于区分早盘、午盘、收盘等不同类型的会话。

### 场景：添加新标签
- **当** 调用工具为当前 Session 添加标签 `morning-session`
- **那么** 标签成功添加到 Session

### 场景：添加多个标签
- **当** 调用工具添加多个标签 `["morning-session", "trading"]`
- **那么** 所有标签成功添加

### 场景：添加已存在的标签
- **当** 添加的标签已存在于 Session
- **那么** 返回成功（幂等操作，不报错）

---

## 需求：Session 识别

支持多种方式识别目标 Session。

### 场景：为当前 Session 添加标签
- **当** 不指定 Session ID
- **那么** 为当前正在进行的 Session 添加标签

### 场景：为指定 Session 添加标签
- **当** 指定 Session ID
- **那么** 为该 Session 添加标签

---

## 需求：标签格式校验

标签必须符合格式要求。

### 场景：拒绝无效标签
- **当** 标签包含特殊字符或空格
- **那么** 返回错误 "invalid tag format"

### 场景：标准化标签
- **当** 标签包含大写字母
- **那么** 自动转换为小写

---

## 需求：预设标签支持

系统预设了特定用途的标签。

### 场景：预设标签列表
- **当** 添加以下预设标签时应有特殊含义：
  - `morning-session`：早盘会话
  - `afternoon-session`：午盘会话
  - `close-session`：收盘会话
- **那么** 系统应识别并正确处理
