# 规格：command-autocomplete

命令自动补全功能规格。

## Purpose

定义 TUI 聊天界面的命令自动补全能力，提升用户输入命令的效率和准确性。

## Requirements

### Requirement: 命令建议显示

系统在用户输入 `/` 开头的命令前缀时，必须显示匹配的命令建议列表。

#### Scenario: 显示命令建议

- **WHEN** 用户在输入框中输入 `/` 开头的文本
- **THEN** 系统显示所有匹配前缀的命令建议
- **AND** 每个建议显示命令名称和描述
- **AND** 默认选中第一个建议
- **AND** 建议列表按命令名称字母排序

#### Scenario: 无匹配命令

- **WHEN** 用户输入的命令前缀没有任何匹配
- **THEN** 不显示建议列表
- **AND** 正常处理后续输入

#### Scenario: 输入空斜杠

- **WHEN** 用户只输入 `/`
- **THEN** 显示所有可用命令
- **AND** 默认选中第一个命令

### Requirement: 命令建议导航

用户必须能够通过键盘在命令建议列表中导航选择。

#### Scenario: 向下移动选择

- **WHEN** 用户按下 `↓` 键
- **THEN** 选中下一个命令建议
- **AND** 如果已在最后一项，保持选中最后一项
- **AND** 视口自动滚动以显示选中项

#### Scenario: 向上移动选择

- **WHEN** 用户按下 `↑` 键
- **THEN** 选中上一个命令建议
- **AND** 如果已在第一项，保持选中第一项
- **AND** 视口自动滚动以显示选中项

#### Scenario: 实时过滤

- **WHEN** 用户继续输入命令前缀
- **THEN** 实时更新匹配的建议列表
- **AND** 重置选中项为第一个匹配项

### Requirement: Tab 自动补全

用户必须能够通过 Tab 键自动补全选中的命令。

#### Scenario: Tab 补全选中的命令

- **WHEN** 用户按下 Tab 键
- **THEN** 输入框内容替换为 `/` + 选中的命令名称 + 空格
- **AND** 光标移动到输入框末尾
- **AND** 关闭建议列表

#### Scenario: 单个匹配时直接补全

- **WHEN** 只有一个匹配的命令
- **THEN** Tab 键直接补全该命令
- **AND** 无需用户选择

#### Scenario: 非命令模式下 Tab 无效

- **WHEN** 输入框内容不是以 `/` 开头
- **THEN** Tab 键不触发补全行为

### Requirement: 命令建议样式

命令建议列表必须有清晰的视觉样式区分选中项。

#### Scenario: 选中项高亮

- **WHEN** 显示命令建议列表
- **THEN** 选中的命令使用高亮样式（前缀 `>` 或不同颜色）
- **AND** 未选中的命令使用普通样式
- **AND** 命令名称和描述使用不同样式区分

#### Scenario: 显示快捷键提示

- **WHEN** 命令建议列表可见
- **THEN** 底部显示快捷键提示：`Tab: 补全 | ↑↓: 选择 | Enter: 执行 | Esc: 取消`
