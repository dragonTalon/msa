# 规格：trade-execution

交易执行能力，提供买入/卖出订单提交、自动成交和交易记录查询功能。

## Purpose

为 MSA 系统提供交易执行能力，用户可以提交买入/卖出订单，系统自动执行成交。所有操作在同一数据库事务中完成，确保数据一致性。交易失败时有明确的错误处理和回滚机制。

## Requirements

### [R-1] 提交买入订单

**场景：正常买入订单**
- **当** 用户提交买入订单
- **且** 账户可用余额充足
- **那么** 创建 PENDING 状态的交易记录
- **且** 从可用余额扣除总金额（数量 × 价格 + 手续费）
- **且** 增加锁定金额
- **且** 立即执行 FillOrder 将状态改为 FILLED
- **且** 释放锁定金额
- **且** 返回交易 ID 和成功提示

**场景：余额不足时拒绝**
- **当** 用户提交买入订单
- **且** 账户可用余额不足
- **那么** 创建 REJECTED 状态的交易记录
- **且** 不扣除任何金额
- **且** 返回友好错误提示（如 "余额不足，需要 XXX.XX 元，可用 XXX.XX 元"）

**场景：自动成交失败时回滚**
- **当** 订单创建成功（PENDING）
- **且** 自动成交操作失败
- **那么** 返回部分成功的错误信息
- **且** 记录交易 ID 便于手动处理
- **且** 保持 PENDING 状态不丢失数据

### [R-2] 提交卖出订单

**场景：正常卖出订单**
- **当** 用户提交卖出订单
- **且** 持仓数量充足
- **那么** 创建 PENDING 状态的交易记录
- **且** 立即执行 FillOrder 将状态改为 FILLED
- **且** 增加可用余额（卖出金额 - 手续费）
- **且** 返回交易 ID 和成功提示

**场景：持仓不足时警告**
- **当** 用户提交卖出订单
- **且** 持仓数量不足
- **那么** 返回错误提示
- **且** 显示当前持仓数量
- **且** 不创建交易记录

**场景：无持仓时拒绝**
- **当** 用户提交卖出订单
- **且** 账户中无该股票的持仓
- **那么** 返回友好错误提示
- **且** 建议用户先查询持仓

### [R-3] 自动成交机制

**场景：买入订单自动成交**
- **当** 买入订单状态为 PENDING
- **那么** 将交易状态更新为 FILLED
- **且** 减少锁定金额（不返还到可用，已在提交时扣除）

**场景：卖出订单自动成交**
- **当** 卖出订单状态为 PENDING
- **那么** 将交易状态更新为 FILLED
- **且** 增加可用余额 = 卖出数量 × 卖出价格 - 手续费

**场景：成交在同一事务内完成**
- **当** 执行自动成交操作
- **那么** 订单创建和成交在同一数据库事务中
- **且** 任何步骤失败都回滚整个操作
- **且** 保证数据一致性

### [R-4] 查询交易记录

**场景：查询所有交易**
- **当** 用户调用交易记录查询 tool
- **且** 不提供任何筛选条件
- **那么** 返回所有交易记录，按时间倒序排列
- **且** 每条记录包含：
  - 交易 ID
  - 股票代码和名称
  - 交易类型（买入/卖出）
  - 数量
  - 价格（元）
  - 金额（元）
  - 手续费（元）
  - 状态
  - 创建时间

**场景：按股票筛选**
- **当** 用户提供股票代码筛选条件
- **那么** 只返回该股票的交易记录

**场景：按类型筛选**
- **当** 用户提供交易类型筛选（BUY/SELL）
- **那么** 只返回该类型的交易记录

**场景：按状态筛选**
- **当** 用户提供状态筛选条件
- **那么** 只返回该状态的交易记录

**场景：限制返回数量**
- **当** 用户指定返回数量限制
- **那么** 只返回最近的 N 条记录
- **且** 默认限制为 50 条防止数据过多

### [R-5] 订单参数处理

**场景：价格单位转换**
- **当** 用户提供的价格为元（float）
- **那么** 转换为毫（int64）存储
- **且** 转换公式：毫 = int64(元 × 100)

**场景：默认手续费**
- **当** 用户未指定手续费
- **那么** 使用默认手续费 5 元
- **且** 转换为 500 毫

**场景：股票名称获取**
- **当** 用户只提供股票代码
- **那么** 通过现有 tool 获取股票名称
- **且** 如果获取失败，使用股票代码作为名称
