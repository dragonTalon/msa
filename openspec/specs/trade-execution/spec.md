# 规格：trade-execution

交易执行能力，提供买入/卖出订单提交、自动成交和交易记录查询功能。

## Purpose

为 MSA 系统提供交易执行能力，用户可以提交买入/卖出订单，系统自动执行成交。所有操作在同一数据库事务中完成，确保数据一致性。交易失败时有明确的错误处理和回滚机制。

## Requirements

### Requirement: 提交买入订单

系统 SHALL 支持用户提交买入订单并自动完成成交。

#### Scenario: 正常买入订单

- **WHEN** 用户提交买入订单
- **AND** 账户可用余额充足
- **THEN** 创建 PENDING 状态的交易记录
- **AND** 从可用余额扣除总金额（数量 × 价格 + 手续费）
- **AND** 增加锁定金额
- **AND** 立即执行 FillOrder 将状态改为 FILLED
- **AND** 释放锁定金额
- **AND** 返回交易 ID 和成功提示

#### Scenario: 余额不足时拒绝

- **WHEN** 用户提交买入订单
- **AND** 账户可用余额不足
- **THEN** 创建 REJECTED 状态的交易记录
- **AND** 不扣除任何金额
- **AND** 返回友好错误提示（如 "余额不足，需要 XXX.XX 元，可用 XXX.XX 元"）

#### Scenario: 自动成交失败时回滚

- **WHEN** 订单创建成功（PENDING）
- **AND** 自动成交操作失败
- **THEN** 返回部分成功的错误信息
- **AND** 记录交易 ID 便于手动处理
- **AND** 保持 PENDING 状态不丢失数据

### Requirement: 提交卖出订单

系统 SHALL 支持用户提交卖出订单并自动完成成交。

#### Scenario: 正常卖出订单

- **WHEN** 用户提交卖出订单
- **AND** 持仓数量充足
- **THEN** 创建 PENDING 状态的交易记录
- **AND** 立即执行 FillOrder 将状态改为 FILLED
- **AND** 增加可用余额（卖出金额 - 手续费）
- **AND** 返回交易 ID 和成功提示

#### Scenario: 持仓不足时警告

- **WHEN** 用户提交卖出订单
- **AND** 持仓数量不足
- **THEN** 返回错误提示
- **AND** 显示当前持仓数量
- **AND** 不创建交易记录

#### Scenario: 无持仓时拒绝

- **WHEN** 用户提交卖出订单
- **AND** 账户中无该股票的持仓
- **THEN** 返回友好错误提示
- **AND** 建议用户先查询持仓

### Requirement: 自动成交机制

系统 SHALL 在订单提交后立即执行自动成交，并在同一事务内完成。

#### Scenario: 买入订单自动成交

- **WHEN** 买入订单状态为 PENDING
- **THEN** 将交易状态更新为 FILLED
- **AND** 减少锁定金额（不返还到可用，已在提交时扣除）

#### Scenario: 卖出订单自动成交

- **WHEN** 卖出订单状态为 PENDING
- **THEN** 将交易状态更新为 FILLED
- **AND** 增加可用余额 = 卖出数量 × 卖出价格 - 手续费

#### Scenario: 成交在同一事务内完成

- **WHEN** 执行自动成交操作
- **THEN** 订单创建和成交在同一数据库事务中
- **AND** 任何步骤失败都回滚整个操作
- **AND** 保证数据一致性

### Requirement: 查询交易记录

系统 SHALL 支持按多种条件查询历史交易记录。

#### Scenario: 查询所有交易

- **WHEN** 用户调用交易记录查询 tool
- **AND** 不提供任何筛选条件
- **THEN** 返回所有交易记录，按时间倒序排列
- **AND** 每条记录包含：
  - 交易 ID
  - 股票代码和名称
  - 交易类型（买入/卖出）
  - 数量
  - 价格（元）
  - 金额（元）
  - 手续费（元）
  - 状态
  - 创建时间

#### Scenario: 按股票筛选

- **WHEN** 用户提供股票代码筛选条件
- **THEN** 只返回该股票的交易记录

#### Scenario: 按类型筛选

- **WHEN** 用户提供交易类型筛选（BUY/SELL）
- **THEN** 只返回该类型的交易记录

#### Scenario: 按状态筛选

- **WHEN** 用户提供状态筛选条件
- **THEN** 只返回该状态的交易记录

#### Scenario: 限制返回数量

- **WHEN** 用户指定返回数量限制
- **THEN** 只返回最近的 N 条记录
- **AND** 默认限制为 50 条防止数据过多

### Requirement: 订单参数处理

系统 SHALL 正确处理订单参数的单位转换和默认值。

#### Scenario: 价格单位转换

- **WHEN** 用户提供的价格为元（float）
- **THEN** 转换为毫（int64）存储
- **AND** 转换公式：毫 = int64(元 × 100)

#### Scenario: 默认手续费

- **WHEN** 用户未指定手续费
- **THEN** 使用默认手续费 5 元
- **AND** 转换为 500 毫

#### Scenario: 股票名称获取

- **WHEN** 用户只提供股票代码
- **THEN** 通过现有 tool 获取股票名称
- **AND** 如果获取失败，使用股票代码作为名称
