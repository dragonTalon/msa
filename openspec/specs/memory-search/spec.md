# 规格：memory-search

> 📚 **知识上下文**：基于模式 003（错误处理策略）- 搜索性能优化

## Purpose

定义跨会话和知识的搜索能力，使用户能够快速找到历史信息。

## Requirements

### Requirement: 跨会话关键词搜索

系统 SHALL 支持在所有历史会话和知识中搜索关键词。

#### Scenario: 简单关键词搜索

- **WHEN** 用户在搜索视图中输入关键词
- **THEN** 同时搜索会话和知识
- **AND** 返回匹配结果列表，按相关性排序
- **AND** 每个结果显示：
  - 类型标签（会话/知识）
  - 标题/摘要
  - 匹配内容片段（高亮关键词）
  - 时间戳
- **AND** 搜索应在 500ms 内完成（使用索引）

#### Scenario: 多关键词搜索

- **WHEN** 用户输入多个关键词（空格分隔）
- **THEN** 搜索包含所有关键词的结果（AND 逻辑）
- **AND** 支持引号包裹精确匹配（如 "腾讯 股价"）
- **AND** 优先显示完全匹配的结果

### Requirement: 股票代码搜索

系统 MUST 支持通过股票代码快速查找相关会话。

#### Scenario: 精确股票代码搜索

- **WHEN** 用户搜索股票代码（如 "00700.HK"）
- **THEN** 显示所有提及该股票的会话
- **AND** 显示相关的关注列表知识
- **AND** 搜索不区分大小写

#### Scenario: 股票名称搜索

- **WHEN** 用户搜索股票名称（如 "腾讯"）
- **THEN** 查找包含该名称的会话
- **AND** 尝试匹配关联的股票代码
- **AND** 同时返回代码和名称匹配的结果

### Requirement: 时间范围过滤

用户 SHALL 能够按时间范围缩小搜索结果。

#### Scenario: 指定日期范围

- **WHEN** 用户设置开始和结束日期
- **THEN** 只搜索该时间范围内的会话和知识
- **AND** 日期支持相对格式：
  - `today`, `yesterday`
  - `7d ago`, `30d ago`
  - `2024-03-01`

#### Scenario: 按月浏览

- **WHEN** 用户选择特定月份
- **THEN** 显示该月的所有会话
- **AND** 按日期分组显示
- **AND** 显示该月的统计信息（会话数、消息数）

### Requirement: 搜索结果分页和导航

搜索结果 MUST 支持分页和键盘导航。

#### Scenario: 分页显示结果

- **WHEN** 搜索结果超过一页（默认 10 条）
- **THEN** 显示当前页码和总页数
- **AND** 支持 PgUp/PgDn 翻页
- **AND** 显示滚动指示器（上方/下方还有 N 条）

#### Scenario: 打开搜索结果

- **WHEN** 用户按 Enter 选择一个搜索结果
- **THEN** 如果是会话，跳转到会话详情
- **AND** 如果是知识，跳转到知识详情
- **AND** 在详情中高亮显示匹配内容

### Requirement: 搜索历史和建议

系统 SHALL 记录搜索历史并提供智能建议。

#### Scenario: 显示搜索历史

- **WHEN** 用户进入搜索视图
- **THEN** 显示最近的 5 个搜索词
- **AND** 支持方向键快速选择历史搜索
- **AND** 按 Enter 重新执行历史搜索

#### Scenario: 搜索建议

- **WHEN** 用户输入搜索关键词
- **THEN** 根据索引提供自动完成建议
- **AND** 建议包括：
  - 历史搜索词
  - 常见股票代码
  - 常见概念名称
- **AND** 按 Tab 接受建议

### Requirement: 高级搜索过滤器

系统 SHALL 支持组合多个搜索条件。

#### Scenario: 按类型过滤

- **WHEN** 用户选择搜索类型（会话/知识/全部）
- **THEN** 只在选定类型中搜索
- **AND** 类型选择应持久化（下次使用记住偏好）

#### Scenario: 按标签过滤

- **WHEN** 用户选择特定标签（如 "技术指标"）
- **THEN** 只显示带有该标签的会话
- **AND** 显示可用的标签列表及其数量

#### Scenario: 组合过滤

- **WHEN** 用户同时设置多个过滤条件
- **THEN** 应用所有过滤条件（AND 逻辑）
- **AND** 清晰显示当前活跃的过滤器
- **AND** 支持快速清除单个或全部过滤器

### Requirement: 搜索性能优化

搜索操作 MUST 使用索引优化，避免全文件扫描。

#### Scenario: 使用内存索引

- **WHEN** 执行搜索操作时
- **THEN** 使用内存索引进行查找
- **AND** 不扫描实际文件内容（除了加载详情时）
- **AND** 索引包含：关键词、股票代码、标签、摘要

#### Scenario: 索引失效重建

- **WHEN** 索引文件损坏或不存在
- **THEN** 自动扫描会话和知识文件重建索引
- **AND** 显示"正在重建索引..."提示
- **AND** 重建完成后自动执行搜索

> 📚 **推荐模式**：使用错误处理策略（pattern 003）
> 搜索失败时回退到基础实现，不应导致整个功能不可用

### Requirement: 搜索结果导出

用户 SHALL 能够导出搜索结果。

#### Scenario: 导出为文本

- **WHEN** 用户选择导出搜索结果
- **THEN** 生成格式化的文本报告
- **AND** 包含：搜索条件、结果摘要、完整内容
- **AND** 保存到用户指定的路径

#### Scenario: 导出为 JSON

- **WHEN** 用户选择导出为 JSON 格式
- **THEN** 生成机器可读的 JSON 文件
- **AND** 包含所有搜索结果的完整数据
- **AND** 支持后续导入或分析
