# 规格：tui-memory-browser

> 📚 **知识上下文**：基于 Bubble Tea TUI 框架和现有的 selector 组件模式

## Purpose

定义记忆浏览器的 TUI 界面，提供用户友好的记忆访问体验。

## Requirements

### Requirement: 记忆主页导航

系统 MUST 提供一个主页作为记忆浏览器的入口点。

#### Scenario: 进入记忆主页

- **WHEN** 用户在聊天界面输入 `/remember`
- **THEN** 切换到记忆主页视图
- **AND** 显示 3 个主要选项：
  - 📖 历史会话
  - 💡 知识库
  - 📊 统计信息
- **AND** 每个选项显示对应的数量（如果有）
- **AND** 支持 1-3 数字快捷键

#### Scenario: 从记忆主页返回聊天

- **WHEN** 用户按 ESC 或 q
- **THEN** 返回到聊天界面
- **AND** 聊天状态保持不变（未丢失输入）

### Requirement: 会话列表视图

系统 MUST 提供可浏览、可搜索的会话列表。

#### Scenario: 显示会话列表

- **WHEN** 用户从主页选择"历史会话"
- **THEN** 显示所有历史会话，按最新优先
- **AND** 每个会话显示：
  - 日期和时间
  - 消息数量
  - 简短摘要（用户第一条输入）
  - 标签
  - 提及的股票代码
- **AND** 支持分页（默认 10 条/页）

#### Scenario: 会话列表搜索

- **WHEN** 用户在会话列表中按 `/` 进入搜索模式
- **THEN** 显示搜索提示符
- **AND** 实时过滤显示匹配的会话
- **AND** 支持按日期、标签、股票代码过滤
- **AND** 按 ESC 退出搜索模式

#### Scenario: 从会话列表查看详情

- **WHEN** 用户按 Enter 选择一个会话
- **THEN** 切换到会话详情视图
- **AND** 传递会话 ID 用于加载完整内容

#### Scenario: 从会话列表删除会话

- **WHEN** 用户按 d 删除选中的会话
- **THEN** 显示确认提示
- **AND** 确认后删除会话文件
- **AND** 更新索引并刷新列表

### Requirement: 会话详情视图

系统 MUST 支持查看完整的历史会话内容。

#### Scenario: 加载会话详情

- **WHEN** 进入会话详情视图
- **THEN** 加载完整的会话 JSON 文件
- **AND** 显示会话元数据：
  - 开始和结束时间
  - 会话时长
  - 使用的模型
  - 提及的股票
- **AND** 显示完整的对话历史
- **AND** 支持滚动浏览

#### Scenario: 会话详情搜索

- **WHEN** 用户在会话详情中按 `/` 搜索
- **THEN** 跳转到第一个匹配的消息
- **AND** 高亮显示匹配内容
- **AND** 支持 n/N 在匹配项之间导航
- **AND** 按 ESC 退出搜索

#### Scenario: 会话详情滚动

- **WHEN** 用户使用方向键或 PgUp/PgDn
- **THEN** 平滑滚动消息列表
- **AND** 显示当前位置（如 5/23）
- **AND** 显示滚动指示器
- **AND** 支持 g 跳到开头，G 跳到结尾

#### Scenario: 会话详情导出

- **WHEN** 用户按 e 导出会话
- **THEN** 生成格式化的文本文件
- **AND** 保存到用户指定的路径
- **AND** 显示成功/失败提示

### Requirement: 知识列表视图

系统 MUST 提供按类型浏览知识的能力。

#### Scenario: 显示知识类型选择

- **WHEN** 用户从主页选择"知识库"
- **THEN** 显示 6 个知识类型：
  - 👤 用户画像
  - 📊 关注列表
  - 📚 学过的概念
  - 📋 交易策略
  - ❓ 常见问答
  - 🔍 其他洞察
- **AND** 每个类型显示数量
- **AND** 支持 1-6 数字快捷键

#### Scenario: 显示特定类型的知识

- **WHEN** 用户选择某个知识类型
- **THEN** 显示该类型的所有知识
- **AND** 每个知识显示：
  - 名称/标题
  - 描述/摘要
  - 提取时间
  - 来源会话
- **AND** 支持分页和过滤

#### Scenario: 查看知识详情

- **WHEN** 用户按 Enter 选择一个知识
- **THEN** 显示完整的知识内容
- **AND** 显示来源会话的链接
- **AND** 如果是概念，显示理解程度

### Requirement: 统计视图

系统 MUST 提供使用统计和趋势分析。

#### Scenario: 显示统计概览

- **WHEN** 用户从主页选择"统计信息"
- **THEN** 显示：
  - 总会话数
  - 总消息数
  - 总知识数
  - 最早和最新会话时间
  - 最常查询的股票 TOP 5
  - 学得最多的概念 TOP 5
- **AND** 使用图表或表格可视化

#### Scenario: 显示使用趋势

- **WHEN** 用户查看统计详情
- **THEN** 按月显示会话数量趋势
- **AND** 按周显示活跃度
- **AND** 显示知识增长曲线

### Requirement: 键盘一致性

所有记忆浏览器视图 MUST 使用一致的键盘快捷键。

#### Scenario: 通用导航键

- **WHEN** 用户在任何记忆视图
- **THEN** 以下快捷键应保持一致：
  - `ESC` - 返回上一级或退出
  - `q` - 退出记忆浏览器（返回聊天）
  - `↑/↓` 或 `j/k` - 上下移动
  - `Enter` - 确认选择
  - `/` - 进入搜索模式
  - `PgUp/PgDn` - 翻页

#### Scenario: 上下文帮助

- **WHEN** 用户在任何视图
- **THEN** 底部显示可用的快捷键
- **AND** 按 `?` 显示详细帮助
- **AND** 帮助包含所有快捷键和功能说明

### Requirement: 视图状态管理

系统 MUST 正确管理视图状态和转换。

#### Scenario: 视图栈管理

- **WHEN** 用户在视图间导航
- **THEN** 维护视图栈（聊天 → 主页 → 列表 → 详情）
- **AND** ESC 返回上一级视图
- **AND** 保留滚动位置和过滤状态
- **AND** 正确处理视图生命周期

#### Scenario: 视图数据刷新

- **WHEN** 用户返回某个视图
- **THEN** 如果数据可能已更新，刷新显示
- **AND** 会话列表应在退出详情后刷新（可能被删除）
- **AND** 知识缓存应在提取新知识后更新

> 📚 **推荐模式**：参考现有的 model_selector.go 实现
> 复用选择器组件和样式，保持 UI 一致性

### Requirement: 错误处理和恢复

系统 MUST 优雅地处理错误情况。

#### Scenario: 文件读取失败

- **WHEN** 尝试读取会话或知识文件失败
- **THEN** 显示友好的错误消息
- **AND** 提供重试选项
- **AND** 不影响其他功能

#### Scenario: 索引损坏

- **WHEN** 索引文件损坏或不存在
- **THEN** 显示"正在重建索引..."
- **AND** 自动扫描文件重建
- **AND** 完成后刷新视图

> 📚 **推荐模式**：使用错误处理策略（pattern 003）
> UI 层面的错误不应导致程序崩溃，应提供清晰的反馈和恢复路径
