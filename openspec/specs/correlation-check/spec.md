## ADDED Requirements

### Requirement: 买入前行业集中度检查

每次买入前必须检查新标的与现有持仓的行业相关性。

行业暴露上限:
- 单一行业 ≤ 总资产 50%
- 单一子行业 ≤ 总资产 30%
- 单一概念 ≤ 总资产 20%

#### Scenario: 同行业持仓超过上限时拒绝买入

- **GIVEN** 现有持仓包含 2 只白酒股（合计占总资产 45%），准备买入第三只白酒股（计划 10% 仓位）
- **WHEN** 买入前检查发现白酒行业暴露将达 55%
- **THEN** 应拒绝买入或要求将仓位降至行业暴露 ≤50%

#### Scenario: 跨行业无冲突时正常通过

- **GIVEN** 现有持仓为消费行业 30%、科技行业 20%，准备买入银行股（金融行业）
- **WHEN** 买入前执行相关性检查
- **THEN** 应正常通过，金融行业暴露将不超过上限

---

### Requirement: 行业分类数据获取

系统必须能获取每只股票的申万行业分类（一级和二级），用于集中度计算。

数据来源: `get_stock_industry` 工具 → 腾讯财经 `stockinfo/jiankuang` API 的 `plate` 字段。

#### Scenario: 查询个股行业分类

- **GIVEN** 股票代码 sh600519（贵州茅台）
- **WHEN** 调用 `get_stock_industry` 工具
- **THEN** 应返回一级行业"食品饮料"(id: 01801120) 和二级行业"白酒Ⅱ"(id: 01801125)

---

### Requirement: 概念板块关联检查

除申万行业外，还需检查概念板块维度的集中度。

- 使用 `get_board_rank` 工具查询概念板块排行
- 使用板块代码查询成分股，检查持仓股票是否同属一个热门概念
- 同一概念板块的持仓暴露 ≤ 总资产 20%

#### Scenario: 检测到概念板块集中风险

- **GIVEN** 持仓 3 只股票均属"半导体"概念板块（pt01801081），合计暴露 35%
- **WHEN** AI 执行持仓概念检查
- **THEN** 应在分析报告中标注"半导体概念集中度 35% 超过 20% 上限，建议分散"
