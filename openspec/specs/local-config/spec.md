# 规格：local-config

本地配置存储能力，支持配置的加载、保存和合并。

## Purpose

管理配置文件的本地存储，支持多源配置的优先级合并，提供灵活的配置加载和缓存机制。

## Requirements

### 需求：环境变量加载

系统 SHALL 从环境变量加载配置。

#### 场景：加载环境变量
- **当** 系统初始化配置时
- **那么** 应检查以下环境变量：
  - `MSA_PROVIDER`
  - `MSA_API_KEY`
  - `MSA_BASE_URL`
  - `MSA_LOG_LEVEL`
  - `MSA_LOG_FILE`
- **并且** 将找到的值合并到配置中

#### 场景：环境变量优先级
- **当** 同一配置项在环境变量和配置文件中都存在
- **那么** 环境变量的值应覆盖配置文件的值

#### 场景：空环境变量处理
- **当** 环境变量设置为空字符串
- **那么** 应视为未设置
- **并且** 使用配置文件或默认值

### 需求：配置优先级合并

系统 SHALL 实现多源配置的优先级合并逻辑。

#### 场景：配置源优先级
- **当** 从多个源加载配置
- **那么** 优先级从高到低应为：
  1. 命令行 `--config` 参数
  2. 环境变量
  3. 配置文件 `~/.msa/msa_config.json`
  4. 默认值

#### 场景：部分覆盖
- **当** 高优先级源只包含部分配置项
- **那么** 未设置的配置项应从低优先级源获取
- **并且** 确保最终配置完整

#### 场景：合并策略
- **当** 合并多个配置源
- **那么** 应按以下策略：
  - 从默认配置开始
  - 应用配置文件值（覆盖默认值）
  - 应用环境变量值（覆盖配置文件值）
  - 应用命令行参数值（覆盖环境变量值）

### 需求：配置验证流程

系统 SHALL 在配置加载后进行验证。

#### 场景：加载后验证
- **当** 配置加载完成
- **那么** 应运行所有配置验证
- **并且** 如果验证失败，记录警告日志
- **并且** 使用默认值替换无效配置

#### 场景：验证失败处理
- **当** 配置验证失败
- **那么** 不应阻止程序启动
- **并且** 应显示友好的错误提示
- **并且** 引导用户使用 `msa config` 修复配置

### 需求：命令行参数解析

系统 SHALL 解析 `--config` 参数。

#### 场景：解析 key=value 格式
- **当** `--config` 参数包含 `=` 符号
- **那么** 应解析为 key-value 对
- **并且** 支持的 key 包括：
  - `provider`
  - `apikey`
  - `baseurl`
  - `loglevel`
  - `logfile`

#### 场景：解析文件路径
- **当** `--config` 参数不包含 `=` 符号
- **并且** 是有效的文件路径
- **那么** 应从该文件加载配置

#### 场景：混合使用
- **当** 多个 `--config` 参数混合使用文件和 key=value
- **那么** 应先加载文件配置
- **然后** 应用 key=value 覆盖

#### 场景：无效参数格式
- **当** `--config` 参数格式无法识别
- **那么** 应显示用法说明
- **并且** 退出并返回错误码

### 需求：配置缓存刷新

系统 SHALL 支持配置的动态重新加载。

#### 场景：清除缓存
- **当** 调用 `ReloadConfig()`
- **那么** 应清除配置缓存
- **并且** 从文件重新加载配置

#### 场景：环境变量重载
- **当** 配置重新加载
- **那么** 应重新读取环境变量
- **并且** 不需要重启程序

### 需求：错误处理改进

系统 MUST 提供改进的错误处理机制。
- **当** 配置文件读取失败（权限、IO 错误）
- **那么** 应记录详细的错误日志
- **并且** 使用默认配置继续运行
- **并且** 向用户显示友好的错误消息

#### 场景：配置文件解析失败
- **当** JSON 解析失败
- **那么** 应记录损坏的配置内容
- **并且** 使用默认配置继续运行
- **并且** 提示用户配置文件已损坏
- **并且** 建议运行 `msa config` 重新配置

#### 场景：配置保存失败
- **当** 配置保存失败
- **那么** 应返回详细的错误信息
- **并且** 包含失败原因（权限、磁盘空间等）
- **并且** TUI 应显示错误并允许重试

### 需求：配置默认值

系统 SHALL 确保所有配置项都有合理的默认值。

#### 场景：Provider 默认值
- **当** 未设置 Provider
- **那么** 应使用 `Siliconflow` 作为默认值

#### 场景：日志配置默认值
- **当** 未设置日志配置
- **那么** 应使用以下默认值：
  - Level: `info`
  - Format: `text`
  - Output: `file`
  - File: `~/.msa/msa.log`
  - ShowColor: `false`
  - TimeFormat: `2006-01-02 15:04:05`

#### 场景：API Key 默认值
- **当** 未设置 API Key
- **那么** 应保持为空字符串
- **并且** 在需要时提示用户配置
